# <a name="use-grunt-in-aspnet-core"></a>Использование Grunt в ASP.NET Core

По [Райс Ноэл](https://blog.falafel.com/falafel-software-recognized-sitefinity-website-year/)

Grunt является запускателя задач JavaScript, автоматизирующего минификации скрипт, компиляции TypeScript, средства «lint» качество кода, предварительной процессоров CSS и практически любые повторяющихся рутинную работу, необходимо, выполнив для поддержки разработки клиентских приложений. Grunt полностью поддерживается в Visual Studio, то, что шаблоны проектов ASP.NET с помощью средства Gulp по умолчанию (см. в разделе [использование Gulp](using-gulp.md)).

В этом примере использует пустой проект ASP.NET Core в качестве начальной точки, демонстрирует, как автоматизировать процесс построения клиента с нуля.

Готовый пример Очищает целевой каталог развертывания, объединяет файлы JavaScript, проверяет качество кода, также, объединяя содержимое файла JavaScript и развертывает в корень веб-приложения. Мы будем использовать следующие пакеты:

* **grunt**: Пакет runner задачи Grunt.

* **grunt-contrib-clean**: Подключаемый модуль, который удаляет файлы или каталоги.

* **grunt-contrib-jshint**: Подключаемый модуль, который просматривает качества кода JavaScript.

* **grunt-contrib-concat**: Подключаемый модуль, который объединяет файлы в один файл.

* **grunt-contrib-uglify**: Подключаемый модуль, который уменьшает JavaScript для уменьшения размера.

* **grunt-contrib-watch**: Подключаемый модуль, который отслеживает действия файла.

## <a name="preparing-the-application"></a>Подготовка приложения

Чтобы начать, настройте новый пустой веб-приложения и добавить файлы TypeScript пример. Файлы TypeScript компилируются в JavaScript с использованием параметров по умолчанию Visual Studio автоматически и будет наш исходный материал для обработки с помощью Grunt.

1.  В Visual Studio создайте новое `ASP.NET Web Application`.

2.  В **новый проект ASP.NET** диалоговое окно, выберите ASP.NET Core **пустой** шаблона и нажмите кнопку "ОК".

3.  В обозревателе решений просмотрите структуру этого проекта. `\src` Папка содержит пустой `wwwroot` и `Dependencies` узлов.

    ![пустой веб-решений](using-grunt/_static/grunt-solution-explorer.png)

4.  Добавьте новую папку с именем `TypeScript` в каталог проекта.

5.  Прежде чем добавить любые файлы, убедитесь, что Visual Studio имеет параметр "компилировать при сохранении" для возвращенных файлов TypeScript. Перейдите к **средства** > **параметры** > **текстовый редактор** > **Typescript**  >  **Проекта**:

    ![Параметры автоматического compliation файлов TypeScript](using-grunt/_static/typescript-options.png)

6.  Щелкните правой кнопкой мыши `TypeScript` каталог и выберите **Добавить > новый элемент** в контекстном меню. Выберите **файл JavaScript** элемента и назовите файл *Tastes.ts* (Примечание \*расширением .ts). Скопируйте строку кода TypeScript ниже в файл (при сохранении, новый *Tastes.js* появится файл с источником JavaScript).
    
    ```typescript
    enum Tastes { Sweet, Sour, Salty, Bitter }
    ```

7.  Добавьте второй файл в **TypeScript** каталог и назовите его `Food.ts`. Скопируйте приведенный ниже код в файл.

    ```typescript
    class Food {
      constructor(name: string, calories: number) {
        this._name = name;
        this._calories = calories;
      }
    
      private _name: string;
      get Name() {
        return this._name;
      }
    
      private _calories: number;
      get Calories() {
        return this._calories;
      }
    
      private _taste: Tastes;
      get Taste(): Tastes { return this._taste }
      set Taste(value: Tastes) {
        this._taste = value;
      }
    }
    ```

## <a name="configuring-npm"></a>Настройка NPM

Настройте NPM для загрузки grunt и grunt задачи.

1. В обозревателе решений щелкните правой кнопкой мыши проект и выберите **Добавить > новый элемент** в контекстном меню. Выберите **файл конфигурации NPM** товара, оставьте имя по умолчанию *package.json*и нажмите кнопку **добавить** кнопки.

2. В *package.json* файл внутри `devDependencies` объекта фигурные скобки, введите «grunt». Выберите `grunt` из Intellisense списка и нажмите клавишу ВВОД. Visual Studio Квота имя пакета grunt и добавьте двоеточие. Справа от двоеточия, выберите последнюю стабильную версию пакета в верхней части списка Intellisense (нажмите клавишу `Ctrl-Space` Если Intellisense не отображается).

    ![grun Intellisense](using-grunt/_static/devdependencies-grunt.png)
    
    > [!NOTE]
    > NPM использует [семантического управления версиями](http://semver.org/) для упорядочения зависимостей. Семантическое управление версиями, также известный как SemVer, определяет пакеты с формирование <major>.<minor>. <patch>. IntelliSense упрощает семантического управления версиями, отображая только несколько распространенных вариантов действий. Верхний элемент в списке Intellisense (0.4.5 в приведенном выше примере) считается последняя стабильная версия пакета. Символ крышки (^) соответствует самой последней основной версии и тильда (~) соответствует самой последней дополнительный номер версии. См. в разделе [NPM semver версии средства синтаксического анализа ссылки](https://www.npmjs.com/package/semver) в соответствии с полной выразительной, предоставляющий SemVer.

3. Добавить дополнительные зависимости, чтобы загрузить grunt-contrib -\* пакеты для *чистой*, *jshint*, *concat*, *uglify*и *watch* как показано в следующем примере. Версии нет необходимости в примере.

    ```json
    "devDependencies": {
      "grunt": "0.4.5",
      "grunt-contrib-clean": "0.6.0",
      "grunt-contrib-jshint": "0.11.0",
      "grunt-contrib-concat": "0.5.1",
      "grunt-contrib-uglify": "0.8.0",
      "grunt-contrib-watch": "0.6.1"
    }
    ```

4. Сохранить *package.json* файла.

Пакеты для каждого элемента devDependencies загрузит вместе с файлами, которые требуются для каждого пакета. Можно найти файлы пакета в `node_modules` каталог, включив **Показать все файлы** кнопка в обозревателе решений.

![grunt node_modules](using-grunt/_static/node-modules.png)

> [!NOTE]
> Если вам нужно, можно вручную восстановить зависимости в обозревателе решений щелкните правой кнопкой мыши `Dependencies\NPM` и выбрав **восстановить пакеты** пункт меню.

![Восстановление пакетов](using-grunt/_static/restore-packages.png)

## <a name="configuring-grunt"></a>Настройка Grunt

Grunt настраивается с помощью манифеста с именем *Gruntfile.js* , определяет, загружает и регистрирует задачи, которые можно запускать вручную или настроить для запуска автоматически в зависимости о событиях в Visual Studio.

1. Щелкните правой кнопкой мыши проект и выберите **Добавить > новый элемент**. Выберите **файл конфигурации Grunt** , то оставьте имя по умолчанию *Gruntfile.js*и нажмите кнопку **добавить** кнопки.

   Исходный код включает определение модуля и `grunt.initConfig()` метод. `initConfig()` Используется для задания параметров для каждого пакета и остальная часть модуля будет загрузить и зарегистрировать задачи.
    
   ```javascript
   module.exports = function (grunt) {
     grunt.initConfig({
     });
   };
   ```

2. Внутри `initConfig()` метод, параметры для добавления `clean` задач, как показано в примере *Gruntfile.js* ниже. Задачу "Очистить" принимает массив строк каталога. Эта задача удаляет файлы из wwwroot/lib и удаляет всю/временный каталог.

    ```javascript
    module.exports = function (grunt) {
      grunt.initConfig({
        clean: ["wwwroot/lib/*", "temp/"],
      });
    };
    ```

3. После метода initConfig(), добавьте вызов `grunt.loadNpmTasks()`. Это сделает задачи запускаемых из Visual Studio.

    ```javascript
    grunt.loadNpmTasks("grunt-contrib-clean");
    ```

4. Сохранить *Gruntfile.js*. Файл должен выглядеть как на снимке экрана ниже.

    ![Начальное gruntfile](using-grunt/_static/gruntfile-js-initial.png)

5. Щелкните правой кнопкой мыши *Gruntfile.js* и выберите **Task Runner Explorer** в контекстном меню. Откроется окно Task Runner Explorer.

    ![меню обозревателя средство выполнения задачи](using-grunt/_static/task-runner-explorer-menu.png)

6. Убедитесь, что `clean` отображается в области **задачи** в Task Runner Explorer.

    ![Список задач explorer средство запуска задач](using-grunt/_static/task-runner-explorer-tasks.png)

7. Щелкните правой кнопкой мыши задачу "Очистить" и выберите **запуска** в контекстном меню. Окно командной строки отображает ход выполнения задачи.

    ![Задача runner explorer выполните задачу "Очистить"](using-grunt/_static/task-runner-explorer-run-clean.png)
    
    > [!NOTE]
    > Нет файлов или каталогов для очистки еще. При желании можно создать их вручную в обозревателе решений и запустите задачу "Очистить" для проверки.
    
8. В методе initConfig(), добавьте запись для `concat` с помощью следующего кода.

    `src` Массива свойств перечислены файлы для объединения, в том порядке, что они должны быть объединены. `dest` Свойство назначает путь объединенный файл, который создается.

    ```javascript
    concat: {
      all: {
        src: ['TypeScript/Tastes.js', 'TypeScript/Food.js'],
        dest: 'temp/combined.js'
      }
    },
    ```
    
    > [!NOTE]
    > `all` Свойство в приведенном выше коде — имя целевого объекта. Целевые объекты используются в некоторые задачи Grunt, чтобы разрешить несколько сред сборки. Можно просмотреть встроенные целевые объекты, с помощью Intellisense или назначить свои собственные.
    
9. Добавление `jshint` задач с помощью следующего кода.

    Программа jshint качества кода выполняется для каждого файла JavaScript, найденные в каталоге temp.
    
    ```javascript
    jshint: {
      files: ['temp/*.js'],
      options: {
        '-W069': false,
      }
    },
    ```

    > [!NOTE]
    > Параметр «-W069» ошибка создается путем jshint при JavaScript используется скобкой синтаксис, чтобы назначить свойство вместо точечной нотации, т. е. `Tastes["Sweet"]` вместо `Tastes.Sweet`. Параметр отключает предупреждение, чтобы разрешить остальная часть процесса, чтобы продолжить.

10. Добавление `uglify` задач с помощью следующего кода.

    Задача уменьшает *combined.js* файл найден во временном каталоге и создает файл результатов в стандартные соглашения об именовании wwwroot/lib  *\<имя файла\>. min.js*.
    
    ```javascript
    uglify: {
     all: {
       src: ['temp/combined.js'],
       dest: 'wwwroot/lib/combined.min.js'
     }
    },
    ```

11. В разделе grunt.loadNpmTasks() вызов, который загружает grunt-contrib-clean включать один и тот же вызов для jshint, concat и uglify, с помощью следующего кода.
    
    ```javascript
    grunt.loadNpmTasks('grunt-contrib-jshint');
    grunt.loadNpmTasks('grunt-contrib-concat');
    grunt.loadNpmTasks('grunt-contrib-uglify');
    ```

12. Сохранить *Gruntfile.js*. Файл должен выглядеть как в примере ниже.

    ![Пример файла полный grunt](using-grunt/_static/gruntfile-js-complete.png)

13. Обратите внимание, что список задач Explorer Runner задача включает в себя `clean`, `concat`, `jshint` и `uglify` задачи. Выполнение каждой задачи в порядке и просмотрите результаты в обозревателе решений. Каждая задача должно работать без ошибок.
    
    ![Диспетчер выполнения задач, выполнять каждую задачу](using-grunt/_static/task-runner-explorer-run-each-task.png)
    
    Создает новую задачу concat *combined.js* файл и помещает его в папку temp. Задача jshint просто выполняется и не создает выходных данных. Создает новую задачу uglify *combined.min.js* файл и помещает его в wwwroot/lib. По завершении решения должен выглядеть примерно как на снимке экрана ниже:
    
    ![в конце концов задачи в обозревателе решений](using-grunt/_static/solution-explorer-after-all-tasks.png)
    
    > [!NOTE]
    > Дополнительные сведения о параметрах для каждого пакета [ https://www.npmjs.com/ ](https://www.npmjs.com/) и lookup имя пакета в поле поиска на главной странице. Например можно выполнять поиск пакет grunt-contrib-clean, чтобы получить ссылку на документацию, объясняющее, все его параметры.

### <a name="all-together-now"></a>Теперь все вместе

Использование Grunt `registerTask()` метод для выполнения ряда задач в определенной последовательности. Например, для выполнения этого примера, действия, описанные в чистой порядок "->" concat "->" jshint "->" uglify, добавьте приведенный ниже код в модуль. Код необходимо добавить как вызовы loadNpmTasks(), за пределами initConfig того же уровня.

```javascript
grunt.registerTask("all", ['clean', 'concat', 'jshint', 'uglify']);
```

Новая задача отображается в Task Runner Explorer под задачи псевдонимов. Можно правой кнопкой мыши и запустите его, так же, как и другие задачи. `all` Задача будет выполняться `clean`, `concat`, `jshint` и `uglify`, в порядке.

![задачи grunt псевдонимов](using-grunt/_static/alias-tasks.png)

## <a name="watching-for-changes"></a>Просмотр изменений

Объект `watch` задач следят над файлами и каталогами. Часы автоматически активирует задачи при обнаружении изменений. Добавьте приведенный ниже код для initConfig для отслеживания изменений для \*JS-файлов в каталоге TypeScript. Если изменяется файл JavaScript, `watch` будет выполняться `all` задачи.

```javascript
watch: {
  files: ["TypeScript/*.js"],
  tasks: ["all"]
}
```

Добавьте вызов `loadNpmTasks()` Показать `watch` задач в Task Runner Explorer.

```javascript
grunt.loadNpmTasks('grunt-contrib-watch');
```

Щелкните правой кнопкой мыши задачу «watch» в Task Runner Explorer и выберите в контекстном меню запуска. Командное окно, показывающий выполняемую задачу Контрольные значения будет отображаться «ожидание...» Сообщение. Откройте один из файлов TypeScript, добавьте пробел и затем сохраните файл. Это будет запустить задачу, Контрольные значения и активируйте другие задачи, чтобы выполняются по порядку. На следующем снимке экрана показан пример запуска.

![под управлением выходных данных задачи](using-grunt/_static/watch-running.png)

## <a name="binding-to-visual-studio-events"></a>Привязка к событиям Visual Studio

Если вам не нужно вручную запустить задачи, каждый раз при работе в Visual Studio, можно привязать задач **перед построить**, **после построения**, **Очистить**, и  **Проект открытым** события.

Следует привязать `watch` таким образом, чтобы он выполняется при каждом запуске Visual Studio открывает. В Task Runner Explorer правой кнопкой мыши задачу контрольных значений и выберите **привязки > Открыть проект** в контекстном меню.

![Привязать задачу к Открытие проекта](using-grunt/_static/bindings-project-open.png)

Выгрузите и перезагрузите проект. Когда снова загружает проект, задачи «watch» запущен автоматически.

## <a name="summary"></a>Сводка

Grunt является запускатель задач мощные, позволяют автоматизировать большинство задач сборки клиента. Grunt использует NPM для доставки пакетов и функций, средств интеграции с Visual Studio. Visual Studio Task Runner Explorer обнаруживает изменения в файлах конфигурации и предоставляет удобный интерфейс для выполнения задач, просмотр выполняющихся задач и привязывание задач к событиям Visual Studio.
