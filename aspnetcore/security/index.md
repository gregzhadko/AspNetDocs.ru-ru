## <a name="authentication-vs-authorization"></a><span data-ttu-id="070ec-110">Проверка подлинности и Авторизация</span><span class="sxs-lookup"><span data-stu-id="070ec-110">Authentication vs. Authorization</span></span>

<span data-ttu-id="070ec-111">Проверка подлинности — это процесс, когда пользователь вводит учетные данные, которые затем сравниваются с данными, хранящимися в операционной системе, базе данных, приложении или ресурсе.</span><span class="sxs-lookup"><span data-stu-id="070ec-111">Authentication is a process in which a user provides credentials that are then compared to those stored in an operating system, database, app or resource.</span></span> <span data-ttu-id="070ec-112">Если они совпадают, пользователи успешно проходят аутентификацию и во время авторизации могут выполнять разрешенные действия.</span><span class="sxs-lookup"><span data-stu-id="070ec-112">If they match, users authenticate successfully, and can then perform actions that they're authorized for, during an authorization process.</span></span> <span data-ttu-id="070ec-113">Авторизация представляет собой процесс, определяющий, какие действия может выполнять пользователь.</span><span class="sxs-lookup"><span data-stu-id="070ec-113">The authorization refers to the process that determines what a user is allowed to do.</span></span>

<span data-ttu-id="070ec-114">Если взглянуть на проверку подлинности с другой стороны, ее можно считать способом входа в определенную область, например на сервер, в базу данных, приложение или ресурс, тогда как авторизация определяет, какие действия с какими объектами может выполнять пользователь в этой области (на сервере, в базе данных или приложении).</span><span class="sxs-lookup"><span data-stu-id="070ec-114">Another way to think of authentication is to consider it as a way to enter a space, such as a server, database, app or resource, while authorization is which actions the user can perform to which objects inside that space (server, database, or app).</span></span>
