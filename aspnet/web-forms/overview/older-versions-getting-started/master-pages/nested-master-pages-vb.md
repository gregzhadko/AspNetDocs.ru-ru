## <a name="the-benefits-of-nested-master-pages"></a><span data-ttu-id="870e7-134">Преимущества вложенных главных страниц</span><span class="sxs-lookup"><span data-stu-id="870e7-134">The Benefits of Nested Master Pages</span></span>

<span data-ttu-id="870e7-135">Многие веб-сайты имеют раздельную структуру сайта, а также более настраиваемые разработки, характерные для определенных типов страниц.</span><span class="sxs-lookup"><span data-stu-id="870e7-135">Many websites have an overarching site design as well as more customized designs specific to certain types of pages.</span></span> <span data-ttu-id="870e7-136">Например, в нашем демонстрационном веб-приложении мы создали раздел "элементарное Администрирование" (страницы в папке `~/Admin`).</span><span class="sxs-lookup"><span data-stu-id="870e7-136">For instance, in our demo web application we have created a rudimentary Administration section (the pages in the `~/Admin` folder).</span></span> <span data-ttu-id="870e7-137">В настоящее время веб-страницы в папке `~/Admin` используют ту же главную страницу, что и страницы, не находящиеся в разделе Администрирование (а именно, `Site.master` или `Alternate.master`в зависимости от выбора пользователя).</span><span class="sxs-lookup"><span data-stu-id="870e7-137">Currently the web pages in the `~/Admin` folder use the same master page as those pages not in the administration section (namely, `Site.master` or `Alternate.master`, depending on the user's selection).</span></span>

> [!NOTE]
> <span data-ttu-id="870e7-138">Сейчас представьте, что наш сайт содержит только одну главную страницу, `Site.master`.</span><span class="sxs-lookup"><span data-stu-id="870e7-138">For now, pretend that our site has just one master page, `Site.master`.</span></span> <span data-ttu-id="870e7-139">Далее в этом руководстве мы будем использовать вложенные главные страницы с двумя (или более) главными страницами, начинающимися с «Использование вложенной главной страницы для администрирования».</span><span class="sxs-lookup"><span data-stu-id="870e7-139">We'll address using nested master pages with two (or more) master pages starting with "Using a Nested Master Page for the Administration Section" later in this tutorial.</span></span>

3. <span data-ttu-id="870e7-144">Создайте новую главную страницу специально для раздела Администрирование, скопируйте разметку из `Site.master`, добавьте сведения о разделе администрирования и ссылки, а затем обновите страницы содержимого в папке `~/Admin`, чтобы использовать эту новую главную страницу.</span><span class="sxs-lookup"><span data-stu-id="870e7-144">Create a new master page specifically for the Administration section, copy over the markup from `Site.master`, add the Administration section-specific information and links, and then update the content pages in the `~/Admin` folder to use this new master page.</span></span>
