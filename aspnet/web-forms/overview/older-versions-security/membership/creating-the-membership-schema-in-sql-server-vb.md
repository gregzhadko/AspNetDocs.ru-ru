## <a name="step-1-deciding-where-to-place-the-user-store"></a><span data-ttu-id="366d1-130">Шаг 1. Выбор места размещения хранилища пользователей</span><span class="sxs-lookup"><span data-stu-id="366d1-130">Step 1: Deciding Where to Place the User Store</span></span>

<span data-ttu-id="366d1-131">Данные приложения ASP.NET обычно хранятся в нескольких таблицах базы данных.</span><span class="sxs-lookup"><span data-stu-id="366d1-131">An ASP.NET application's data is commonly stored in a number of tables in a database.</span></span> <span data-ttu-id="366d1-132">При реализации схемы базы данных `SqlMembershipProvider` необходимо решить, следует ли поместить схему членства в ту же базу данных, что и данные приложения, или в альтернативную базу данных.</span><span class="sxs-lookup"><span data-stu-id="366d1-132">When implementing the `SqlMembershipProvider` database schema we must decide whether to place the Membership schema in the same database as the application data or in an alternate database.</span></span>
