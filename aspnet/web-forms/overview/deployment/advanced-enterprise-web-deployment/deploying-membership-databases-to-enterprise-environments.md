## <a name="what-are-the-issues-when-you-deploy-a-membership-database"></a><span data-ttu-id="269a5-111">Каковы проблемы при развертывании базы данных членства?</span><span class="sxs-lookup"><span data-stu-id="269a5-111">What Are the Issues When You Deploy a Membership Database?</span></span>

<span data-ttu-id="269a5-112">В большинстве случаев при создании стратегии развертывания для базы данных сначала необходимо учитывать, какие данные необходимо развернуть.</span><span class="sxs-lookup"><span data-stu-id="269a5-112">In most cases, when you devise a deployment strategy for a database, the first thing you need to consider is what data you want to deploy.</span></span> <span data-ttu-id="269a5-113">В среде разработки или тестирования может потребоваться развернуть данные учетной записи пользователя, чтобы упростить и быстро выполнить тестирование.</span><span class="sxs-lookup"><span data-stu-id="269a5-113">In a development or test environment, you might want to deploy user account data to facilitate quick and easy testing.</span></span> <span data-ttu-id="269a5-114">В промежуточной или рабочей среде крайне маловероятно, чтобы развертывать данные учетной записи пользователя.</span><span class="sxs-lookup"><span data-stu-id="269a5-114">In a staging or production environment, it's very unlikely that you'd want to deploy user account data.</span></span>

<span data-ttu-id="269a5-115">К сожалению, базы данных членства ASP.NET представляют некоторые конкретные проблемы, которые делают это решение гораздо более сложным:</span><span class="sxs-lookup"><span data-stu-id="269a5-115">Unfortunately, ASP.NET membership databases introduce some specific challenges that make this decision a lot more complex:</span></span>

- <span data-ttu-id="269a5-116">При развертывании только схемы база данных членства остается в нерабочем состоянии.</span><span class="sxs-lookup"><span data-stu-id="269a5-116">A schema-only deployment will leave the membership database in a non-operational state.</span></span> <span data-ttu-id="269a5-117">Это связано с тем, что база данных членства содержит некоторые данные конфигурации (в таблице **aspnet\_счемаверсионс** ), необходимые для работы базы данных.</span><span class="sxs-lookup"><span data-stu-id="269a5-117">This is because the membership database includes some configuration data (in the **aspnet\_SchemaVersions** table) that the database requires in order to function.</span></span> <span data-ttu-id="269a5-118">Таким образом, если вы выполняете развертывание только схемы базы данных членства, чтобы исключить данные учетной записи пользователя, необходимо запустить скрипт, выполняемый после развертывания, чтобы добавить необходимые данные конфигурации.</span><span class="sxs-lookup"><span data-stu-id="269a5-118">As such, if you perform a schema-only deployment of your membership database in order to exclude user account data, you'll need to run a post-deployment script to add the essential configuration data.</span></span>
- <span data-ttu-id="269a5-119">В зависимости от настройки базы данных членства поставщик членства может использовать ключ компьютера для шифрования паролей и сохранения их в базе данных.</span><span class="sxs-lookup"><span data-stu-id="269a5-119">Depending on how your membership database is configured, the membership provider may use the machine key to encrypt passwords and store them in the database.</span></span> <span data-ttu-id="269a5-120">В этом случае любые данные учетной записи пользователя, развертываемые с помощью базы данных, станут непригодными для использования на целевом сервере.</span><span class="sxs-lookup"><span data-stu-id="269a5-120">In this case, any user account data you deploy with the database will become unusable on the destination server.</span></span> <span data-ttu-id="269a5-121">По этой причине развертывание данных учетной записи пользователя не является поддерживаемым сценарием.</span><span class="sxs-lookup"><span data-stu-id="269a5-121">For this reason, deploying user account data is not a supported scenario.</span></span>

## <a name="choosing-a-membership-database-strategy"></a><span data-ttu-id="269a5-122">Выбор стратегии базы данных членства</span><span class="sxs-lookup"><span data-stu-id="269a5-122">Choosing a Membership Database Strategy</span></span>

<span data-ttu-id="269a5-123">Используйте эти рекомендации при выборе способа предоставления базы данных членства в среде Enterprise Server.</span><span class="sxs-lookup"><span data-stu-id="269a5-123">Use these guidelines when you choose how to provision a membership database in an enterprise server environment:</span></span>

- <span data-ttu-id="269a5-124">Везде, где это возможно, не развертывайте базы данных членства.</span><span class="sxs-lookup"><span data-stu-id="269a5-124">Wherever possible, do not deploy membership databases.</span></span> <span data-ttu-id="269a5-125">Вместо этого создайте базу данных членства вручную на целевом сервере базы данных.</span><span class="sxs-lookup"><span data-stu-id="269a5-125">Instead, create the membership database manually on the target database server.</span></span> <span data-ttu-id="269a5-126">Если вы еще не настроили схему базы данных членства, можно просто создать новую в месте в месте назначения с помощью [средства ASP.NET SQL Server Registration Tool (aspnet\_регскл. exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span><span class="sxs-lookup"><span data-stu-id="269a5-126">If you haven't customized your membership database schema, you can simply create a new one in situ at the destination using the [ASP.NET SQL Server Registration Tool (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span></span>
- <span data-ttu-id="269a5-127">Если вы не имеете возможности развернуть базу данных&#x2014;членства, например, если вы внесли значительные изменения в схему&#x2014;базы данных, то следует выполнить развертывание только схемы базы данных членства, чтобы исключить данные учетной записи пользователя, а затем запустить скрипт, выполняемый после развертывания, чтобы добавить необходимые данные конфигурации.</span><span class="sxs-lookup"><span data-stu-id="269a5-127">If you have no option but to deploy a membership database&#x2014;for example, if you've made extensive modifications to the database schema&#x2014;you should perform a schema-only deployment of the membership database, to exclude user account data, and then run a post-deployment script to add any required configuration data.</span></span> <span data-ttu-id="269a5-128">Общие рекомендации по этим подходам см. в статье [развертывание базы данных членства ASP.NET без включения учетных записей пользователей](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span><span class="sxs-lookup"><span data-stu-id="269a5-128">You can find broad guidance on these approaches in [How to: Deploy the ASP.NET Membership Database Without Including User Accounts](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span></span>

<span data-ttu-id="269a5-129">Важно помнить, что *схема базы данных членства, скорее всего, будет довольно статической*.</span><span class="sxs-lookup"><span data-stu-id="269a5-129">It's important to remember that *the schema of your membership database is likely to be fairly static*.</span></span> <span data-ttu-id="269a5-130">Даже если вы настроили базу данных членства, вряд ли придется обновлять схему регулярно,&#x2014;она не будет изменяться с той же частотой, что и код в веб-приложении или проекте базы данных.</span><span class="sxs-lookup"><span data-stu-id="269a5-130">Even if you've customized the membership database, it's unlikely that you'll need to update the schema on a regular basis&#x2014;it's not going to change with the same frequency as the code in a web application or a database project.</span></span> <span data-ttu-id="269a5-131">Таким образом, не следует включать базу данных членства в автоматизированные или одноэтапные процессы развертывания.</span><span class="sxs-lookup"><span data-stu-id="269a5-131">As such, you shouldn't need to include the membership database in any automated or single-step deployment processes.</span></span>

## <a name="using-vsdbcmd-to-update-a-membership-database-schema"></a><span data-ttu-id="269a5-132">Использование программы VSDBCMD для обновления схемы базы данных членства</span><span class="sxs-lookup"><span data-stu-id="269a5-132">Using VSDBCMD to Update a Membership Database Schema</span></span>

<span data-ttu-id="269a5-133">При изменении структуры базы данных членства после первого развертывания может потребоваться не использовать средство веб-развертывания службы IIS (IIS) (веб-развертывание) для повторного развертывания базы данных.</span><span class="sxs-lookup"><span data-stu-id="269a5-133">If you modify the structure of your membership database after your first deployment, you may not want to use the Internet Information Services (IIS) Web Deployment Tool (Web Deploy) to redeploy the database.</span></span> <span data-ttu-id="269a5-134">Функции развертывания базы данных в веб-развертывание не включают возможность вносить разностные обновления в целевую базу данных&#x2014;, веб-развертывание должны удалять и повторно создавать базу данных.</span><span class="sxs-lookup"><span data-stu-id="269a5-134">The database deployment functionality in Web Deploy doesn't include the capability to make differential updates to a destination database&#x2014;instead, Web Deploy must drop and re-create the database.</span></span> <span data-ttu-id="269a5-135">Это означает, что теряются все существующие данные учетной записи пользователя, которые обычно нежелательны в промежуточной или рабочей среде.</span><span class="sxs-lookup"><span data-stu-id="269a5-135">This means that you lose any existing user account data, which is typically undesirable in staging or production environments.</span></span>

<span data-ttu-id="269a5-136">Альтернативой является использование служебной программы VSDBCMD для обновления схемы целевой базы данных.</span><span class="sxs-lookup"><span data-stu-id="269a5-136">The alternative is to use the VSDBCMD utility to update the schema of your destination database.</span></span> <span data-ttu-id="269a5-137">VSDBCMD включает две важные возможности.</span><span class="sxs-lookup"><span data-stu-id="269a5-137">VSDBCMD includes two important capabilities.</span></span> <span data-ttu-id="269a5-138">Сначала он может импортировать схему существующей базы данных в DBSCHEMA-файл.</span><span class="sxs-lookup"><span data-stu-id="269a5-138">First, it can import the schema of an existing database into a .dbschema file.</span></span> <span data-ttu-id="269a5-139">Во-вторых, он может развернуть DBSCHEMA-файл в существующей базе данных в качестве разностного обновления.
