# <a name="creating-a-team-project-in-tfs"></a>Создание командного проекта в Team Foundation Server

по [Джейсон Lee](https://github.com/jrjlee)

[Загрузить PDF-файл](https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/63/56/8130.DeployingWebAppsInEnterpriseScenarios.pdf)

> В этом разделе описывается создание нового командного проекта в Team Foundation Server (TFS) 2010.


Этот раздел является частью серии учебников, исходя из требования к развертыванию enterprise вымышленной компании Fabrikam, Inc. В этой серии руководств используется пример решения&#x2014; [решения диспетчера контактов](../web-deployment-in-the-enterprise/the-contact-manager-solution.md)&#x2014;для представления веб-приложения с более реалистичные уровень сложности, включая приложения ASP.NET MVC 3, Windows Communication Служба Foundation (WCF) и проект базы данных.

## <a name="task-overview"></a>Общие сведения о задачах

Чтобы подготовить и использовать новый командный проект в TFS, необходимо выполнить следующие действия:

- Предоставить разрешения пользователю, которому будет создание нового командного проекта.
- Создайте командный проект.
- Предоставить разрешения членам команды, которые будут работать над проектом.
- Проверьте в некоторое содержимое.

В этом разделе показано, как для выполнения этих процедур, и будет выполнен поиск пользователей и роли заданий, которые могут отвечать за каждой процедуры. Имейте в виду, что, в зависимости от структуры вашей организации, каждый из этих задач может быть ответственность за другое лицо.

Задачи и пошаговые руководства в этом разделе предполагается, что вы установили и настроили TFS и что вы создали коллекцию командных проектов как часть процесса настройки. Дополнительные сведения на следующих предположениях и дополнительные общие сведения о сценарии, см. в разделе [настроить сервер сборки TFS для развертывания веб-](configuring-a-tfs-build-server-for-web-deployment.md).

## <a name="grant-permissions-to-the-team-project-creator"></a>Предоставление разрешений для создатель командного проекта

Для создания нового командного проекта, потребуются следующие разрешения:

- Необходимо иметь **создавать новые проекты** разрешение на уровне приложений TFS. Обычно предоставить это разрешение, добавив пользователям **Администраторы коллекции проектов** группу TFS. **Администраторы Team Foundation** глобальной группы также включает это разрешение.
- Необходимо иметь разрешение на создание новых сайтов групп внутри семейства узлов SharePoint, соответствующий коллекции командных проектов TFS. Обычно предоставьте это разрешение, при добавлении пользователя в группу SharePoint с **полный контроль** семейство веб-сайтов права на сайте SharePoint.
- Если вы используете SQL Server Reporting Services функции, необходимо быть членом **Диспетчер содержимого Team Foundation** роли в службах Reporting Services.

### <a name="who-performs-these-procedures"></a>Кто выполняет эти процедуры?

Как правило пользователь или группа, являющегося администратором развертывания TFS также выполняет эти процедуры.

Так как это более высоким уровнем привилегий набора разрешений, новые командные проекты обычно создаются с небольшой набор пользователей, ответственных за администрирование развертывания TFS. Разработчики не обычно получают разрешения, необходимые для создания новых командных проектов.

### <a name="grant-permissions-in-tfs"></a>GRANT, предоставление разрешений в Team Foundation Server

Если вы хотите разрешить пользователю создавать командные проекты, первой высокого уровня задачей является добавление пользователю **Администраторы коллекции проектов** группы для коллекции командных проектов.

**Чтобы добавить пользователя в группу администраторов коллекции проектов**

1. На сервере TFS на **запустить** последовательно выберите пункты **все программы**, нажмите кнопку **Microsoft Team Foundation Server 2010**, а затем нажмите кнопку **Team Foundation Консоль администрирования**.
2. В представлении в виде дерева навигации разверните **уровень приложений**, а затем нажмите кнопку **коллекций командных проектов**.

    ![](creating-a-team-project-in-tfs/_static/image1.png)
3. В **коллекций командных проектов** панели, выберите коллекцию, вы хотите управлять командных проектов.

    ![](creating-a-team-project-in-tfs/_static/image2.png)
4. На **Общие** щелкните **членство в группе**.

    ![](creating-a-team-project-in-tfs/_static/image3.png)
5. В **глобальные группы** выберите **Администраторы коллекции проектов** группу и нажмите **свойства**.
6. В **свойства группы Team Foundation Server** диалоговом окне выберите **Windows пользователь или группа**, а затем нажмите кнопку **добавить**.

    ![](creating-a-team-project-in-tfs/_static/image4.png)
7. В **Выбор пользователей, компьютеров или групп** диалоговое окно, введите имя пользователя, который вы хотите иметь возможность создавать командные проекты, щелкните **проверить имена**, а затем нажмите кнопку **ОК** .

    ![](creating-a-team-project-in-tfs/_static/image5.png)
8. В **свойства группы Team Foundation Server** диалоговом окне щелкните **ОК**.
9. В **глобальные группы** диалоговом окне щелкните **закрыть**.

### <a name="grant-permissions-in-sharepoint-services"></a>Предоставление разрешений в службах SharePoint

Далее необходимо предоставить пользователю разрешение на создание новых сайтов групп в коллекции сайтов SharePoint, соответствующий вашей коллекции командных проектов TFS.

**Чтобы предоставить разрешения полного доступа для коллекции сайтов SharePoint**

1. В консоли администрирования Team Foundation Server на **коллекций командных проектов** выберите коллекции командных проектов, которым требуется управлять.
2. На **сайт SharePoint** вкладке, обратите внимание на значение **текущее местоположение сайта по умолчанию** URL-адрес.

    ![](creating-a-team-project-in-tfs/_static/image6.png)
3. Откройте Internet Explorer, а затем перейдите на URL-адрес, вы записали на шаге 2.

    > [!NOTE]
    > Если не выполнен вход на для Windows от имени пользователя, создавшего коллекцию командных проектов, необходимо войти в SharePoint от имени этого пользователя для продолжения.
4. На **действия сайта** меню, щелкните **параметры сайта**.

    ![](creating-a-team-project-in-tfs/_static/image7.png)
5. На **параметры сайта** раздела **пользователями и разрешениями**, нажмите кнопку **пользователи и группы**.
6. На панели навигации слева щелкните элемент **группы**.

    ![](creating-a-team-project-in-tfs/_static/image8.png)
7. На **пользователи и группы: Все группы** щелкните **настроить группы для этого сайта**.

    ![](creating-a-team-project-in-tfs/_static/image9.png)

   > [!NOTE]
   > Может появиться <strong>HTTP 404 не найдено</strong> ошибка из-за ошибки кодирования double HTTP. В этом случае замените URL-адрес с этим:   
   > `[site_collection_URL]/_layouts/permsetup.aspx` Например:  
   > `http://tfs/sites/Fabrikam%20Web%20Projects/_layouts/permsetup.aspx` 
8. На **настроить группы для этого сайта** странице, пользователь, который будет создавать командные проекты, чтобы добавить **владельцев** группу и нажмите **ОК**.

    ![](creating-a-team-project-in-tfs/_static/image10.png)

Дополнительные сведения о включении пользователям создавать командные проекты в коллекции командных проектов см. в разделе [Установка прав администратора для коллекций командных проектов](https://msdn.microsoft.com/library/dd547204.aspx).

## <a name="create-a-new-team-project-and-add-users"></a>Создание нового командного проекта и добавление пользователей

При наличии необходимых разрешений можно использовать **Team Explorer** окно в Visual Studio 2010 для создания нового командного проекта. Такой подход предоставляет мастер, который собирает все необходимые данные и выполняет необходимые задачи в TFS, SharePoint и SQL Server Reporting Services. Также необходимо предоставить разрешения для нового командного проекта участникам команды разработчиков, чтобы включить их для добавления и изменения содержимого.

### <a name="who-performs-these-procedures"></a>Кто выполняет эти процедуры?

Обычно администратор TFS или руководитель группы разработчиков выполнение этих процедур.

### <a name="create-a-new-team-project"></a>Создание нового командного проекта

Далее описано, как создать новый командный проект в Team Foundation Server 2010.

**Для создания нового командного проекта**

1. На **запустить** последовательно выберите пункты **все программы**, нажмите кнопку **Microsoft Visual Studio 2010**, щелкните правой кнопкой мыши **Microsoft Visual Studio 2010**, а затем нажмите кнопку **Запуск от имени администратора**.

    > [!NOTE]
    > Если не выполнить Visual Studio 2010 с правами администратора, на последний шаг не удастся мастеру создания командного проекта.
2. Если **контроль учетных записей пользователей** появится диалоговое окно, нажмите кнопку **Да**.
3. В Visual Studio на **Team** меню, щелкните **подключиться к Team Foundation Server**.

    > [!NOTE]
    > Если вы уже настроили подключение к серверу TFS, можно пропустить шаги 4 – 7.
4. В **подключение к командному проекту** диалоговом окне щелкните **серверов**.
5. В **Добавление или удаление Team Foundation Server** диалоговом окне щелкните **добавить**.
6. В **добавить Team Foundation Server** диалоговом окне укажите сведения об экземпляре TFS, а затем нажмите кнопку **ОК**.

    ![](creating-a-team-project-in-tfs/_static/image11.png)
7. В **Добавление или удаление Team Foundation Server** диалоговом окне щелкните **закрыть**.
8. В **подключение к командному проекту** диалоговом окне выберите коллекцию, вы хотите добавить и нажмите кнопку проектов экземпляре TFS, вы хотите подключиться, выберите команду **Connect**.

    ![](creating-a-team-project-in-tfs/_static/image12.png)
9. В **Team Explorer** окно, щелкните правой кнопкой мыши командного проекта коллекции, а затем нажмите кнопку **нового командного проекта**.

    ![](creating-a-team-project-in-tfs/_static/image13.png)
10. В **нового командного проекта** диалоговом окне укажите имя и описание для командного проекта и нажмите кнопку **Далее**.

    > [!NOTE]
    > Если командный проект содержит пробелы, вы можете столкнуться некоторые проблемы при переходе используйте средство Internet Information Services (IIS) веб-развертывания (веб-развертывание) для развертывания пакетов из выходной путь. Пробелы в пути можно сделать гораздо труднее — для выполнения команд на веб-развертывания.

    ![](creating-a-team-project-in-tfs/_static/image14.png)
11. На **Выбор шаблона процесса** выберите шаблон процесса, который вы хотите использовать для управления процессом разработки и нажмите кнопку **Далее**.

    > [!NOTE]
    > Дополнительные сведения о шаблонах процессов для TFS, см. в разделе [шаблоны процессов и инструменты](https://msdn.microsoft.com/vstudio/aa718795).
12. На **параметры сайта группы** странице, оставьте без изменений параметры по умолчанию и нажмите кнопку **Далее**.
13. Этот параметр создает или определяет, сайт группы SharePoint, который связан с командным проектом TFS. Ваша команда разработчиков может использовать этот сайт для управления документацией, участвовать в беседах, создание вики-страниц и выполнять различные задачи, которые не связаны с кодом. Дополнительные сведения см. в разделе [взаимодействий между продуктов SharePoint и Team Foundation Server](https://msdn.microsoft.com/library/ms253177.aspx).
14. На **укажите параметры системы управления версиями** странице, оставьте без изменений параметры по умолчанию и нажмите кнопку **Далее**.
15. Этот параметр определяет или создает расположение в иерархии папок TFS, который будет выступать в качестве корневой папки для содержимого.
16. На **подтверждение настроек командного проекта** щелкните **Готово**.
17. Когда новый командный проект создан успешно, на **командный проект создан** щелкните **закрыть**.

### <a name="add-users-to-a-team-project"></a>Добавление пользователей в командный проект

Теперь, после создания нового командного проекта, можно предоставить разрешения для пользователей, чтобы включить их, чтобы начать добавление и совместная работа над содержимым.

**Чтобы добавить пользователей в командный проект**

1. В Visual Studio 2010 в **Team Explorer** окно, щелкните правой кнопкой мыши командный проект, выберите пункт **параметры командного проекта**, а затем нажмите кнопку **членство в группе**.

    ![](creating-a-team-project-in-tfs/_static/image15.png)
2. Чтобы пользователи могут добавлять, изменять и удалять код, в системе управления версиями, необходимо добавить его к **участники** группы.
3. В **группы проекта** выберите **участники** группу и нажмите **свойства**.

    ![](creating-a-team-project-in-tfs/_static/image16.png)
4. В **свойства группы Team Foundation Server** диалоговом окне выберите **Windows пользователь или группа**, а затем нажмите кнопку **добавить**.

    ![](creating-a-team-project-in-tfs/_static/image17.png)
5. В **Выбор пользователей, компьютеров или групп** диалоговое окно, введите имя пользователя, который вы хотите добавить в командный проект, нажмите кнопку **проверить имена**, а затем нажмите кнопку **ОК**.

    ![](creating-a-team-project-in-tfs/_static/image18.png)
6. В **свойства группы Team Foundation Server** диалоговом окне щелкните **ОК**.
7. В **группы проекта** диалоговом окне щелкните **закрыть**.

## <a name="conclusion"></a>Заключение

На этом этапе новый командный проект готов к использованию, и ваша команда разработчиков можно начать добавление содержимого и совместная работа на процесс разработки.
