## <a name="creating-the-project"></a>Создание проекта

1. Запустите Visual Studio.
2. В меню **файл** в Visual Studio выберите **создать проект** . 

    ![Создание элемента меню "проект — создать проект"](create-the-project/_static/image1.png)
3. В левой части выберите **шаблоны** -&gt;  **C# Visual** -&gt; группы **веб-** шаблонов.
4. Выберите шаблон **веб-приложение ASP.NET** в центральном столбце.  
 В этой серии руководств используется .NET Framework 4.5.2.
5. Присвойте проекту имя *WingtipToys* и нажмите кнопку **ОК** . 

    ![Создание проекта — диалоговое окно "Создание проекта"](create-the-project/_static/image2.png)

    > [!NOTE]
    > Имя проекта в этой серии руководств — **WingtipToys**. Рекомендуется использовать это *точное* имя проекта, чтобы код, предоставленный во всей серии руководств, был работает должным образом.

6. Нажмите кнопку **изменить проверку подлинности** . Выберите **учетные записи отдельных пользователей** и нажмите кнопку **ОК** .

7. Выберите шаблон **веб-формы** и нажмите кнопку **ОК** .

    ![Создание проекта — новый шаблон проекта](create-the-project/_static/image3.png)

Создание проекта займет немного времени. Когда оно будет готово, откройте страницу **Default. aspx** .

![Создание проекта — новый шаблон проекта](create-the-project/_static/image4.png)

Можно переключиться между представлением **конструктора** и представлением **исходного кода** , выбрав параметр в нижней части центрального окна. В режиме **конструктора** отображаются ASP.NET веб-страницы, главные страницы, страницы содержимого, HTML-страницы и пользовательские элементы управления с помощью представления NEAR-WYSIWYG. Представление **исходного кода** отображает разметку HTML для веб-страницы, которую можно изменить.

> [!TIP] 
> 
> **Основные сведения о платформах ASP.NET**
> 
> Веб-формы ASP.NET позволяют создавать динамические веб-сайты с использованием обычной функции перетаскивания и модели на основе событий. Область конструирования и сотни элементов управления и компонентов позволяют быстро создавать комплексные сайты с пользовательским интерфейсом и доступом к данным. Магазин Wingtip Toy основан на веб-формах ASP.NET, но многие из концепций, которые вы узнаете в этой серии руководств, применимы ко всем ASP.NET.
> 
> ASP.NET предлагает четыре основные платформы разработки:
> 
> - [Веб-формы ASP.NET](../../../index.md)  
>  Платформа веб-форм предназначена для разработчиков, которые предпочитают декларативное программирование и управление на основе управления, например Microsoft Windows Forms (WinForms) и WPF/XAML/Silverlight. Она предлагает модель разработки, управляемую конструктором WYSIWYG, поэтому она популярна для разработчиков, которые ищут среду быстрой разработки приложений (RAD) для веб-разработки. Если вы не работали с веб-программированием и знакомы с традиционными средствами разработки клиентских средств Microsoft RAD (например, для Visual Basic и C#Visual), вы можете быстро создать веб-приложение, не имея возможности HTML и JavaScript.
> - [ASP.NET MVC](../../../../mvc/index.md)  
>  ASP.NET MVC предназначен для разработчиков, заинтересованных в шаблонах и принципах, таких как разработка на основе тестирования, разделение проблем, инверсия управления (IoC) и внедрение зависимостей (DI). Эта инфраструктура способствует отделению уровня бизнес-логики веб – приложения от уровня представления.
> - [Веб-страницы ASP.NET](../../../../web-pages/index.md)  
>  Веб-страницы ASP.NET предназначены для разработчиков, которым требуется простая статья о веб-разработке, а также строки PHP. В модели веб-страниц создаются HTML-страницы, а затем на страницу добавляется серверный код, позволяющий динамически управлять отображением разметки. Веб-страницы специально разработаны как облегченная платформа, и она является самой простой точкой входа в ASP.NET для людей, знающих HTML, но не имеющих широких возможностей программирования, например учащихся или любителей. Это также хороший способ для веб-разработчиков, знающих PHP или аналогичные платформы для начала использования ASP.NET.
> - [Одностраничное приложение ASP.NET](../../../../single-page-application/index.md)  
>  ASP.NET одностраничное приложение (SPA) позволяет создавать приложения, включающие значительные взаимодействия на стороне клиента, используя HTML 5, CSS 3 и JavaScript. Обновление ASP.NET and Web Tools 2012,2 поставляется с новым шаблоном для создания одностраничных приложений с помощью выколачивание. js и веб-API ASP.NET. Помимо нового шаблона SPA, новые шаблоны SPA, созданные сообществом, также доступны для загрузки.
> 
> В дополнение к четырем основным платформам разработки ASP.NET также предлагает дополнительные технологии, которые важны для понимания и знакомы с, но не рассматриваются в этой серии руководств:
> 
> - [Веб-API ASP.NET](../../../../web-api/index.md) — платформа для создания служб HTTP, которые достигают широкого спектра клиентов, включая браузеры и мобильные устройства.
> - [ASP.NET SignalR](../../../../signalr/index.md) — Библиотека, которая упрощает разработку веб-функций в режиме реального времени.

## <a name="aspnet-web-forms-background"></a>Фон веб-форм ASP.NET

ASP.NET Web Forms — это страницы, основанные на технологии Microsoft ASP.NET, в которой код, выполняемый на сервере, динамически создает выходные данные веб-страницы в браузере или клиентском устройстве. Страница веб-форм ASP.NET автоматически подготавливает соответствующий браузер HTML для таких функций, как стили, макет и т. д. Веб-формы совместимы с любым языком, поддерживаемым средой CLR .NET, например Microsoft Visual Basic и Microsoft Visual C#. Кроме того, веб-формы строятся на основе [платформы Microsoft .NET](https://msdn.microsoft.com/vstudio/aa496123), которая предоставляет такие преимущества, как управляемая среда, безопасность типов и наследование.

При выполнении страницы веб-форм ASP.NET страница проходит по жизненному циклу, в котором выполняется последовательность действий по обработке. Эти шаги включают инициализацию, создание экземпляров элементов управления, восстановление и поддержание состояния, выполнение кода обработчика событий и отрисовку. По мере знакомства с возможностями ASP.NET Web Forms важно понимать [жизненный цикл страницы ASP.NET](https://msdn.microsoft.com/library/ms178472(v=vs.100).aspx) , чтобы можно было писать код на соответствующем этапе жизненного цикла для желаемого результата.

Когда веб-сервер получает запрос на страницу, он находит страницу, обрабатывает ее, отправляет в браузер, а затем удаляет всю информацию о странице. Если пользователь снова запрашивает ту же страницу, сервер повторяет всю последовательность, повторно обменяя страницу с нуля. Другими словами, на сервере не должно быть памяти страниц, которые он обрабатывает — страницы не имеют состояния. Платформа ASP.NET Pages автоматически обрабатывает задачу обслуживания состояния страницы и ее элементов управления, а также предоставляет явные способы поддержания состояния сведений, относящихся к приложению.

> [!TIP] 
> 
> **Функции веб-приложений в шаблоне приложения Web Forms**
> 
> Шаблон приложения веб-форм ASP.NET предоставляет широкий набор встроенных функций. Она не только предоставляет вам страницу *Home. aspx* , страницу *About. aspx* , страницу *Contact. aspx* , но также включает функции членства, которые регистрируют пользователей и сохраняют свои учетные данные, чтобы они могли входить на веб-сайт. В этом обзоре содержатся дополнительные сведения о некоторых функциях, содержащихся в шаблоне приложения ASP.NET Web Forms, и о том, как они используются в приложении Wingtip Toys.
> 
> **Членство**
> 
> [ASP.NET](https://msdn.microsoft.com/library/yh26yfzy.aspx) Удостоверение сохраняет учетные данные пользователей в базе данных, созданной приложением. Когда пользователи входят в систему, приложение проверяет свои учетные данные, считывая базу данных. В папке *учетной записи* вашего проекта содержатся файлы, реализующие различные компоненты членства: регистрация, вход, изменение пароля и авторизация доступа. Кроме того, веб-формы ASP.NET поддерживают OAuth и OpenID Connect. Эти улучшения проверки подлинности позволяют пользователям входить на сайт с использованием существующих учетных данных, таких как Facebook, Twitter, Windows Live и Google.
> 
> ![Создание проекта — обозреватель решений (ASP.NET Identity)](create-the-project/_static/image7.png)
> 
> По умолчанию шаблон создает базу данных членства, используя имя базы данных по умолчанию на экземпляре SQL Server Express LocalDB, сервер базы данных разработки, поставляемый с Visual Studio Express 2013 для Web.
> 
> **SQL Server Express LocalDB**
> 
> [SQL Server Express LocalDB](https://technet.microsoft.com/library/hh510202.aspx) — это упрощенная версия SQL Server, которая обладает многими программируемыми функциями базы данных SQL Server. SQL Server Express LocalDB выполняется в пользовательском режиме и имеет быструю установку без настройки, которая содержит краткий список необходимых компонентов для установки. В Microsoft SQL Server любой код базы данных или Transact-SQL можно переместить из SQL Server Express LocalDB в SQL Server и SQL Azure без каких-либо действий по обновлению. Таким образом, SQL Server Express LocalDB можно использовать в качестве среды разработки для приложений, предназначенных для всех выпусков SQL Server. SQL Server Express LocalDB позволяет использовать такие функции, как хранимые процедуры, определяемые пользователем функции и агрегаты, .NET Framework интеграцию, пространственные типы и другие, недоступные в SQL Server Compact.
> 
> **Главные страницы**
> 
> [Главная страница ASP.NET](https://msdn.microsoft.com/library/wtxbf3hh.aspx) определяет единообразный внешний вид и поведение для всех страниц в приложении. Макет главной страницы объединяется с содержимым страницы отдельного содержимого для создания окончательной страницы, которую видит пользователь. В приложении Wingtip Toys вы изменяете главную страницу *site. master* так, чтобы все страницы на веб-сайте Wingtip Toys совместно совпадали с логотипом и панелью навигации.
> 
> **HTML5**
> 
> Шаблон приложения веб-форм ASP.NET поддерживает [HTML5](http://www.w3schools.com/html/html5_intro.asp), который является последней версией языка разметки HTML. HTML5 поддерживает новые элементы и функциональные возможности, упрощающие создание веб-сайтов.
> 
> **Modernizr**
> 
> Для браузеров, которые не поддерживают HTML5, можно использовать [Modernizr](http://www.modernizr.com/). Modernizr — это библиотека JavaScript с открытым исходным кодом, которая может определять, поддерживает ли браузер функции HTML5, и включать их, если это не так. В шаблоне приложения веб-форм ASP.NET Modernizr устанавливается как пакет NuGet.
> 
> **Начальная загрузка**
> 
> В шаблонах проектов Visual Studio 2013 используется [Начальная](http://getbootstrap.com/)Загрузка, макет и инфраструктура, созданные Twitter. Начальная загрузка использует CSS3 для обеспечения быстроты реагирования, что означает возможность динамического адаптации макетов к различным размерам окон браузера. Вы также можете воспользоваться функцией начальной загрузки, чтобы упростить изменение внешнего вида и поведения приложения. По умолчанию шаблон веб-приложения ASP.NET в Visual Studio 2013 включает начальную загрузку в качестве пакета NuGet.
> 
> **Пакеты NuGet**
> 
> Шаблон приложения веб-форм ASP.NET включает набор пакетов [NuGet](http://www.nuget.org/) . Эти пакеты предоставляют функциональные возможности в виде библиотек и средств с открытым исходным кодом. Существует множество пакетов, которые помогут вам создавать и тестировать приложения. Visual Studio упрощает добавление, удаление и обновление пакетов NuGet. Разработчики могут также создавать и добавлять пакеты в NuGet.
> 
> ![Создание проекта — диалоговое окно "NuGet"](create-the-project/_static/image8.png)
> 
> При установке пакета NuGet копирует файлы в решение и автоматически выполняет все необходимые изменения, например добавляя ссылки и изменяя конфигурацию, связанную с веб-приложением. Если вы решили удалить библиотеку, NuGet удалит файлы и отменяет все изменения, внесенные в проект, чтобы не было ничего ненужным. NuGet доступен в меню **Сервис** в Visual Studio.
> 
> **jQuery**
> 
> [jQuery](http://jquery.com/) — это быстрая и краткая библиотека JavaScript, которая упрощает обход документов HTML, обработку событий, анимацию и взаимодействие AJAX для быстрой разработки веб-приложений. Библиотека JavaScript jQuery включена в шаблон приложения веб-форм ASP.NET в виде пакета NuGet.
> 
> **Ненавязчивая проверка**
> 
> Встроенные проверяющие элементы управления настроены на использование ненавязчивого кода JavaScript для логики проверки на стороне клиента. Это значительно сокращает объем кода JavaScript, отображаемого в разметке страницы, и уменьшает общий размер страницы. Ненавязчивая проверка добавляется глобально в шаблон приложения веб-форм ASP.NET на основе параметра в элементе &lt;appSettings&gt; файла *Web. config* в корне приложения.
> 
> **Entity Framework Code First**
> 
> Помимо функций в шаблоне приложения ASP.NET Web Forms, приложение Wingtip Toys использует [Entity Framework Code First](https://weblogs.asp.net/scottgu/archive/2010/12/08/announcing-entity-framework-code-first-ctp5-release.aspx), который является библиотекой NuGet, которая обеспечивает разработку на основе кода при работе с данными. Поместив его просто, он создает часть приложения базы данных на основе написанного кода. Используя Entity Framework, вы получаете данные и управляете ими как строго типизированные объекты. Это позволяет сосредоточиться на бизнес-логике в приложении, а не на деталях доступа к данным.
> 
> Дополнительные сведения об установленных библиотеках и пакетах, входящих в состав шаблона веб-форм ASP.NET, см. в списке установленных пакетов NuGet. Для этого в Visual Studio создайте новый проект Web Forms, выберите **инструменты** > **диспетчер пакетов NuGet** > **управления пакетами NuGet для решения**и выберите **установленные пакеты** в диалоговом окне **Управление пакетами NuGet** .
