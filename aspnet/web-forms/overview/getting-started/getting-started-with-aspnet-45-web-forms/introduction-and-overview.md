# <a name="getting-started-with-aspnet-45-web-forms-and-visual-studio-2017"></a>Приступая к работе с ASP.NET 4.5 и Visual Studio 2017


[Скачайте пример проекта Wingtip Toys (C#)](http://go.microsoft.com/fwlink/?LinkID=389434&clcid=0x409) или [скачайте электронную книгу (PDF)](http://download.microsoft.com/download/0/F/B/0FBFAA46-2BFD-478F-8E56-7BF3C672DF9D/Getting%20Started%20with%20ASP.NET%204.5%20Web%20Forms%20and%20Visual%20Studio%202013.pdf)

В этой серии руководств показано, как создать приложение веб-форм ASP.NET с ASP.NET 4.5 и Microsoft Visual Studio 2017. 

## <a name="introduction"></a>Вступление

В этой серии руководств, чтобы руководство по созданию приложения веб-форм ASP.NET с помощью Visual Studio 2017 и ASP.NET 4.5. Вы создадите приложение с именем **Wingtip Toys** — упрощенный веб-сайт Интернет-магазин на продаже товаров через Интернет. Во время ряда выделяются новые функции ASP.NET 4.5.

### <a name="target-audience"></a>Целевая аудитория

Разработчики знакомы с веб-форм ASP.NET, целевая аудитория для этой серии руководств.

Вам необходимо иметь некоторые знания в следующих областях:

- Объектно ориентированное программирование (ООП) и языков
- Веб-разработки (HTML, CSS, JavaScript)
- реляционные базы данных
- N-уровневой архитектуре

Для просмотра этих областей, рассмотрите возможность изучения следующим содержимым:

- [Приступая к работе с Visual C#](https://msdn.microsoft.com/library/a72418yk.aspx)
- [Разработка веб-](https://msdn.microsoft.com/beginner/bb308760.aspx), [HTML, CSS, JavaScript, SQL, PHP, JQuery](http://w3schools.com/)
- [Реляционная база данных](http://en.wikipedia.org/wiki/Relational_database)
- [Многоуровневая архитектура](http://en.wikipedia.org/wiki/Multitier_architecture)

### <a name="application-features"></a>Компоненты приложений

Функции веб-форма ASP.NET, представленные в этой серии:

- Проект веб-приложения (не проект веб-сайта)
- Веб-формы
- Главные страницы, конфигурации
- Начальная загрузка
- Платформа Entity Framework Code First, LocalDB
- Проверка запросов
- Элементы управления данные со строгой типизацией
- Привязка модели
- Заметки к данным
- Поставщики значений
- SSL и OAuth
- ASP.NET Identity, конфигурации и авторизации
- Малозаметная проверка
- Маршрутизация
- Обработка ошибок ASP.NET

### <a name="application-scenarios-and-tasks"></a>Задачи и сценариях приложений

Следующие задачи серии руководств.

- Создание, проверка и выполнение проекта
- Создание структуры базы данных
- Инициализация и заполнение базы данных
- Настройка пользовательского интерфейса с помощью стилей, графики и главной страницы
- Добавление страниц и элементов навигации
- Отображение меню сведений и данные о продуктах
- Создание корзины для покупок
- Поддержка добавления SSL и OAuth
- Добавление метода оплаты
- Включая роль администратора и пользователя в приложение
- Ограничение доступа к определенным страницам и папки
- Отправка файла веб-приложения
- Реализация проверки входных данных
- Регистрация маршрутов для веб-приложения
- Реализация обработки ошибок и ведения журнала ошибок

## <a name="overview"></a>Обзор

В этой серии руководств, предназначен для кого-то знакомы основные понятия программирования, но создавать новые для веб-форм ASP.NET. Если вы уже знакомы с веб-форм ASP.NET, в этой серии по-прежнему помогут вам узнать о новых возможностях ASP.NET 4.5. Читатели знакомы с программирования основные понятия и веб-форм ASP.NET, см. Дополнительные руководства веб-форм, в [Приступая к работе](../../../index.md) разделе на веб-сайте ASP.NET.

ASP.NET 4.5, приведенные в этой серии руководств включает следующие компоненты:

- Простой пользовательский Интерфейс для создания проектов, которая предлагает [поддержку многих платформ ASP.NET](../../../../visual-studio/overview/2013/creating-web-projects-in-visual-studio.md#add) (веб-форм, MVC и веб-API).
- [Начальная загрузка](../../../../visual-studio/overview/2013/creating-web-projects-in-visual-studio.md#bootstrap), макет, темы и framework гибкий Дизайн.
- [ASP.NET Identity](../../../../identity/index.md), новая система членства ASP.NET, который работает так же, во всех платформах ASP.NET и работает с веб-размещения программного обеспечения, отличные от IIS.
- [Entity Framework 6](https://msdn.microsoft.com/data/ef.aspx)

  Обновление для платформы Entity Framework, что позволяет:
  - Извлечения и манипулирования данными в виде строго типизированных объектов
  - Асинхронно получить доступ к данным
  - Обрабатывать временные сбои соединений
  - Журнал инструкций SQL

Полный список компонентов ASP.NET 4.5, см. в разделе [ASP.NET and Web Tools для заметки о выпуске Visual Studio 2013](../../../../visual-studio/overview/2013/release-notes.md).

## <a name="prerequisites"></a>Предварительные требования

Перед началом работы убедитесь, что на компьютере установлено следующее программное обеспечение:

- [Microsoft Visual Studio 2017 или Microsoft Visual Studio Community 2017](https://visualstudio.microsoft.com/downloads/).

Платформа .NET Framework устанавливается автоматически.

В этой серии руководств используется Microsoft Visual Studio Community 2017. Можно использовать либо, или Microsoft Visual Studio 2017 для завершения этой серии руководств.

Обратите внимание на следующие особенности Visual Studio:

* Microsoft Visual Studio 2017 и Microsoft Visual Studio Community 2017, называются *Visual Studio* данной серии учебных курсов.

* Visual Studio 2017 устанавливается рядом со все более ранние версии уже установлена. Узлы, созданные в более ранних версиях можно открывать в Visual Studio 2017 и в предыдущих версиях по-прежнему.

* При первом запуске Visual Studio, предполагается, вы выбрали *веб-разработки* параметры. Дополнительные сведения см. в разделе [Как Выберите параметры среды веб-разработки](https://msdn.microsoft.com/library/ff521558.aspx).

После установки необходимых компонентов, вы будете готовы приступить к созданию веб-проекта, представленные в этой серии руководств.
