В этом руководстве показано, как обновить пример приложения Wingtip Toys, чтобы добавить пользовательскую роль и использовать ASP.NET Identity. Здесь также показано, как реализовать страницу администрирования, из которой пользователь с пользовательской ролью может добавлять и удалять продукты с веб-сайта.

[ASP.NET Identity](../../../../identity/overview/getting-started/introduction-to-aspnet-identity.md) — это система членства, используемая для создания веб-приложения ASP.NET и доступная в ASP.NET 4,5. ASP.NET Identity используется в шаблоне проекта веб-форм Visual Studio 2013, а также в шаблонах для [одностраничного приложения](../../../../single-page-application/index.md) [ASP.NET MVC](../../../../mvc/index.md), [веб-API ASP.NET](../../../../web-api/index.md)и ASP.NET. Можно также установить ASP.NET Identity систему с помощью NuGet, когда вы начинаете с пустого веб-приложения. Однако в этой серии руководств используется **веб-форма**projecttemplate, которая включает в себя ASP.NET Identityную систему. ASP.NET Identity позволяет легко интегрировать данные пользовательского профиля с данными приложений. Кроме того, ASP.NET Identity позволяет выбирать модель сохраняемости для профилей пользователей в приложении. Данные можно хранить в базе данных SQL Server или в другом хранилище данных, включая хранилища данных *NoSQL* , такие как таблицы хранилища Windows Azure.

> [!NOTE] 
> 
> Обратите внимание, что при создании объекта `RoleStore` используется универсальный тип `IdentityRole`. Это означает, что `RoleStore` может содержать только объекты `IdentityRole`. Кроме того, с помощью универсальных ресурсов ресурсы в памяти обрабатываются лучше.

Затем объект `RoleManager` создается на основе только что созданного объекта `RoleStore`. Объект `RoleManager` предоставляет API, связанный с ролью, который можно использовать для автоматического сохранения изменений в `RoleStore`. `RoleManager` может содержать только объекты `IdentityRole`, поскольку в коде используется универсальный тип `<IdentityRole>`.

Чтобы определить, существует ли роль "canEdit" в базе данных членства, вызовите метод `RoleExists`. Если это не так, то создается роль.

Создание `UserManager`ного объекта кажется более сложным, чем элемент управления `RoleManager`, однако это почти то же самое. Он закодируется только на одной строке, а не в нескольких. Здесь передаваемый параметр создается как новый объект, содержащийся в круглых скобках.

### <a name="how-the-application-will-work"></a>Как будет работать приложение

Когда пользователь, принадлежащий к роли "canEdit", переходит на страницу в первый раз, `DropDownAddCategory`элемент управления **DropDownList** заполняется, как описано выше. Элемент управления **DropDownList** `DropDownRemoveProduct`также заполняется продуктами, использующими тот же подход. Пользователь, принадлежащий роли "canEdit", выбирает тип категории и добавляет сведения о продукте (**имя**, **Описание**, **цену**и **файл изображения**). JPEG*или *. jpg*). Затем файл изображения сохраняется в папку примера приложения Wingtip Toys. Затем новый продукт добавляется в базу данных. Для добавления нового продукта создается новый экземпляр класса `AddProducts` и имена Products. ## <a name="running-the-application"></a>Запуск приложения

Теперь можно запустить приложение, чтобы увидеть, как можно добавлять, удалять и обновлять элементы в корзине для покупок. Итоговая корзина для покупок будет отражать общую стоимость всех товаров в корзине для покупок.

1. В обозреватель решений нажмите клавишу **F5** , чтобы запустить пример приложения Wingtip Toys.  
   Откроется браузер и отобразится страница *Default. aspx* .
2. Щелкните ссылку **войти в** верхней части страницы. 

    ![Членство и администрирование — вход в систему](membership-and-administration/_static/image2.png)

   Отобразится страница *Login. aspx* .
3. Используйте следующие имя пользователя и пароль:  
   Имя пользователя: canEditUser@wingtiptoys.com  
   Пароль: PA $ $word 1 

    ![Членство и администрирование — страница входа](membership-and-administration/_static/image3.png)
4. Нажмите кнопку **Вход в** нижней части страницы.
5. В верхней части следующей страницы щелкните ссылку **администратора** , чтобы вернуться на страницу *админпаже. aspx* . 

    ![Членство и администрирование — ссылка администратора](membership-and-administration/_static/image4.png)
6. Чтобы проверить входную проверку, нажмите кнопку " **Добавить продукт** ", не добавляя сведения о продукте. 

    ![Членство и администрирование — страница "Администрирование"](membership-and-administration/_static/image5.png)

   Обратите внимание, что отображаются обязательные сообщения поля.
7. Добавьте сведения для нового продукта, а затем нажмите кнопку **Добавить продукт** . 

    ![Членство и администрирование — Добавление продукта](membership-and-administration/_static/image6.png)
8. Выберите **продукты** в меню навигации вверху, чтобы просмотреть добавленный новый продукт. 

    ![Членство и администрирование — показывать новый продукт](membership-and-administration/_static/image7.png)
9. Щелкните ссылку **Admin (администратор** ), чтобы вернуться на страницу администрирования.
10. В разделе **удалить продукт** на странице выберите новый продукт, добавленный в **дропдовнлистбокс**.
11. Нажмите кнопку **удалить продукт** , чтобы удалить новый продукт из приложения. 

    ![Членство и администрирование — удаление продукта](membership-and-administration/_static/image8.png)
12. В верхнем меню навигации выберите **продукты** , чтобы убедиться, что продукт был удален.
13. Нажмите кнопку Выход, чтобы **Подключиться** к существующему режиму администрирования.   
    Обратите внимание, что в верхней области навигации больше не отображается пункт меню **Администрирование** .
