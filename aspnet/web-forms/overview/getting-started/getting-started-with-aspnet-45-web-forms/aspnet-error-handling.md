## <a name="overview"></a>Обзор

Приложения ASP.NET, необходима возможность обработки ошибок, возникающих во время выполнения, согласованным способом. ASP.NET использует среда CLR (CLR), который предоставляет способ уведомления приложений об ошибках унифицированным способом. При возникновении ошибки, создается исключение. Исключением является любой ошибки, условие или неожиданное поведение, которое обнаруживает приложения.

В .NET Framework исключение — это объект, наследуемый от класса `System.Exception`. Исключение создается из области кода, где произошла проблема. Исключение передается вверх по стеку вызовов в место, где приложение предоставляет код для обработки исключения. Если приложение не обрабатывает исключение, браузер принудительно отобразить сведения об ошибке.

Рекомендуется, обработки ошибок в на уровне кода в `Try` / `Catch` / `Finally` блоки в коде. Попробуйте разместить эти блоки, таким образом, пользователь может устранить проблемы в контексте, в котором они расположены. Если слишком далеко, из которой произошла ошибка в блоках обработки ошибок, становится все сложнее предоставлять пользователям информацию, необходимую для устранения проблемы.

### <a name="exception-class"></a>Класс исключения

Класс исключений является базовым классом, от которого наследуются исключения. Большинство объектов исключения являются экземплярами некоторого класса, производного класса исключения, такие как `SystemException` класс, `IndexOutOfRangeException` класс, или `ArgumentNullException` класса. Класс Exception имеет свойства, такие как `StackTrace` свойство, `InnerException` свойство и `Message` свойство, предоставляют определенные сведения о возникшей ошибки.

### <a name="exception-inheritance-hierarchy"></a>Иерархия наследования исключений

Среда выполнения имеет базовый набор исключений, производных от `SystemException` класс, который среда выполнения создает исключение при обнаружении исключения. Большинство классов, унаследованных от класса Exception, такие как `IndexOutOfRangeException` класс и `ArgumentNullException` класса, не реализуют дополнительные элементы. Таким образом наиболее важные сведения для исключения можно найти в иерархии исключений, имени исключения и информации, содержащейся в исключении.

### <a name="exception-handling-hierarchy"></a>Иерархия обработки исключений

В приложении веб-форм ASP.NET исключения могут обрабатываться на основе обработка в иерархии. Исключения могут обрабатываться на следующих уровнях:

- Уровень приложения
- Уровень страницы
- Уровень кода

Когда приложение обрабатывает исключения, Дополнительные сведения об исключении, которое наследуется от класса Exception можно часто извлекается и отображается для пользователя. В дополнение к приложений, страницы и уровне кода также может обрабатывать исключения на уровне модуля HTTP и с помощью пользовательского обработчика IIS.

### <a name="safe-error-messages"></a>Безопасные сообщения об ошибках

Это **важно отметить** , когда приложение отображает сообщения об ошибках, он не должны содержать сведения, которые пользователь-злоумышленник может помочь атаковать приложение. Например если приложение попытка записи базу данных, следует отображает сообщение об ошибке, включающее имя пользователя, им. По этой причине для пользователя отображается сообщение об общей ошибке красным цветом. Все дополнительную информацию об ошибке отображаются только для разработчика на локальном компьютере.

## <a name="using-elmah"></a>С помощью ELMAH

ELMAH (модули ведения журнала ошибок и обработчики) — это средство ведения журнала ошибок, который можно подключить к приложению ASP.NET, как пакет NuGet. ELMAH предоставляет следующие возможности:

- Ведение журнала необработанных исключений.
- Веб-страницы, чтобы просмотреть весь журнал записи необработанных исключений.
- Веб-страницы, чтобы просмотреть подробные сведения о каждом в журнал исключение.
- Уведомление по электронной почте каждой ошибки во время, когда оно возникает.
- RSS-канал последние 15 ошибок из журнала.

Прежде чем можно будет работать с помощью ELMAH, необходимо установить его. Это легко с помощью *NuGet* пакет установщика. Как упоминалось ранее в этой серии руководств, NuGet — это расширение Visual Studio, которое облегчает установку и обновление библиотек с открытым кодом и средства в Visual Studio.

1. В Visual Studio из **средства** меню, выберите **диспетчер пакетов NuGet** > **управление пакетами NuGet для решения**. 

    ![Обработка ошибок ASP.NET — управление пакетами NuGet для решения](aspnet-error-handling/_static/image6.png)
2. **Управление пакетами NuGet** в Visual Studio отображается диалоговое окно.
3. В **управление пакетами NuGet** диалогового окна последовательно раскройте элементы **Online** слева, а затем выберите **nuget.org**. Затем найти и установить **ELMAH** пакет из списка доступных пакетов через Интернет. 

    ![Обработка ошибок ASP.NET — пакет NuGet ELMA](aspnet-error-handling/_static/image7.png)
4. Необходимо иметь подключение к Интернету для загрузки пакета.
5. В **выберите проекты** диалогового окна поле, убедитесь, что **WingtipToys** выбора выбран и нажмите кнопку **ОК**. 

    ![Обработка ошибок ASP.NET — проекты диалоговое окно выбора](aspnet-error-handling/_static/image8.png)
6. Нажмите кнопку **закрыть** в **управление пакетами NuGet** диалоговое окно, при необходимости.
7. Если Visual Studio запрашивает перезагрузить все открытые файлы, выберите "**Да для всех**«.
8. Пакет ELMAH добавляет записи для себя в *Web.config* файл в корне проекта. Если Visual Studio появится необходимость перезагрузки измененный *Web.config* щелкните **Да**.

ELMAH теперь готов для хранения необработанных возникающие ошибки.

### <a name="viewing-the-elmah-log"></a>Просмотр журнала ELMAH

Просмотр журнала ELMAH несложно, но сначала вы создадите необработанное исключение, который будет записываться в журнал ELMAH.

1. Нажмите клавишу **CTRL + F5** запустить пример приложения Wingtip Toys.
2. Для записи необработанного исключения в журнал ELMAH, перейдите в браузере следующий URL-адрес (используя свой номер порта):  
    `https://localhost:44300/NoPage.aspx` Отображается страница ошибки.
3. Чтобы отобразить журнал ELMAH, перейдите в браузере следующий URL-адрес (используя свой номер порта):  
    `https://localhost:44300/elmah.axd`
