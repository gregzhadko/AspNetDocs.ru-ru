## <a name="overview"></a>Обзор

Приложения ASP.NET должны уметь обрабатывать ошибки, происходящие во время выполнения, согласованным образом. ASP.NET использует среду CLR, которая предоставляет способ уведомления приложений об ошибках единообразно. При возникновении ошибки возникает исключение. Исключением является любая ошибка, условие или непредвиденное поведение, которое встречается в приложении.

В .NET Framework исключение — это объект, наследуемый от класса `System.Exception`. Исключение создается из области кода, где произошла проблема. Исключение передается вверх по стеку вызовов в место, где приложение предоставляет код для решения исключения. Если приложение не обрабатывает исключение, браузер вынужден отобразить сведения об ошибке.

Рекомендуется выполнять обработку ошибок на уровне кода в `Try`/`Catch`/`Finally` блоков в коде. Попытайтесь разместить эти блоки, чтобы пользователь мог исправить проблемы в контексте, в котором они происходят. Если блоки обработки ошибок находятся далеко от того места, где произошла ошибка, больше сложно предоставить пользователям сведения, необходимые для решения проблемы.

### <a name="exception-class"></a>Класс Exception

Класс Exception является базовым классом, от которого наследуются исключения. Большинство объектов исключений — это экземпляры некоторого производного класса класса Exception, например класс `SystemException`, класс `IndexOutOfRangeException` или класс `ArgumentNullException`. Класс Exception имеет такие свойства, как свойство `StackTrace`, свойство `InnerException` и свойство `Message`, предоставляющие конкретные сведения о произошедшей ошибке.

### <a name="exception-inheritance-hierarchy"></a>Иерархия наследования исключений

Среда выполнения имеет базовый набор исключений, производных от класса `SystemException`, который среда выполнения создает при возникновении исключения. Большинство классов, которые наследуют от класса Exception, например класс `IndexOutOfRangeException` и класс `ArgumentNullException`, не реализуют дополнительные члены. Поэтому наиболее важные сведения об исключении можно найти в иерархии исключений, имени исключения и информации, содержащейся в исключении.

### <a name="exception-handling-hierarchy"></a>Иерархия обработки исключений

В приложении ASP.NET Web Forms исключения могут обрабатываться на основе определенной иерархии обработки. Исключение может быть обработано на следующих уровнях:

- Уровень приложения
- Уровень страницы
- Уровень кода

Когда приложение обрабатывает исключения, дополнительные сведения об исключении, унаследованном от класса Exception, часто могут быть получены и отображены пользователю. Помимо уровня приложения, страницы и кода, можно также выполнять обработку исключений на уровне модуля HTTP и с помощью пользовательского обработчика IIS.

### <a name="code-level-error-handling"></a>Обработка ошибок на уровне кода

Оператор try-catch состоит из блока try, за которым следует одно или несколько предложений catch, указывающих обработчики для различных исключений. При возникновении исключения среда CLR ищет оператор catch, который обрабатывает это исключение. Если выполняемый в данный момент метод не содержит блок catch, среда CLR просматривает метод, который вызвал текущий метод, и так далее в стеке вызовов. Если блок catch не найден, среда CLR выводит пользователю сообщение о необработанном исключении и останавливает выполнение программы.

### <a name="safe-error-messages"></a>Сообщения об ошибках в безопасности

**Важно отметить** , что когда приложение отображает сообщения об ошибках, оно не должно предоставлять сведения о том, что злонамеренный пользователь может помочь в атаке приложения. Например, если приложение неудачно пытается выполнить запись в базу данных, оно не должно отображать сообщение об ошибке, содержащее имя пользователя, которое оно использует. По этой причине пользователю отображается общее сообщение об ошибке красного цвета. Все дополнительные сведения об ошибке отображаются только для разработчика на локальном компьютере.

## <a name="using-elmah"></a>Использование ELMAH

ELMAH (модули ведения журнала ошибок и обработчики) — это средство регистрации ошибок, которое подключается к приложению ASP.NET в качестве пакета NuGet. ELMAH предоставляет следующие возможности:

- Ведение журнала необработанных исключений.
- Веб-страница для просмотра всего журнала перекодированных необработанных исключений.
- Веб-страница для просмотра всех сведений о каждом исключении из журнала.
- Уведомление по электронной почте о каждой ошибке во время ее возникновения.
- RSS-канал последних 15 ошибок из журнала.

Прежде чем можно будет работать с ELMAH, необходимо установить его. Это просто с помощью установщика пакетов *NuGet* . Как упоминалось ранее в этой серии руководств, NuGet — это расширение Visual Studio, которое упрощает установку и обновление библиотек и средств с открытым исходным кодом в Visual Studio.

1. В Visual Studio в меню **Сервис** выберите **диспетчер пакетов NuGet** > **Управление пакетами NuGet для решения**. 

    ![Обработка ошибок ASP.NET — управление пакетами NuGet для решения](aspnet-error-handling/_static/image6.png)
2. Диалоговое окно **Управление пакетами NuGet** отображается в Visual Studio.
3. В диалоговом окне **Управление пакетами NuGet** в левой части окна разверните узел в **сети** , а затем выберите **NuGet.org**. Затем найдите и установите пакет **ELMAH** из списка доступных пакетов в сети. 

    ![Обработка ошибок ASP.NET — пакет NuGet ЕЛМА](aspnet-error-handling/_static/image7.png)
4. Для скачивания пакета необходимо подключение к Интернету.
5. В диалоговом окне **Выбор проектов** убедитесь, что выбран параметр **WingtipToys** , и нажмите кнопку **ОК**. 

    ![Обработка ошибок ASP.NET — диалоговое окно выбора проектов](aspnet-error-handling/_static/image8.png)
6. При необходимости нажмите кнопку **Закрыть** в диалоговом окне **Управление пакетами NuGet** .
7. Если Visual Studio запрашивает перезагрузку всех открытых файлов, выберите "**Да для всех**".
8. Пакет ELMAH добавляет записи для себя в файл *Web. config* в корне проекта. Если Visual Studio запросит, хотите ли вы перезагрузить измененный файл *Web. config* , нажмите кнопку **Да**.

Теперь ELMAH готов к хранению любых необработанных ошибок.

### <a name="viewing-the-elmah-log"></a>Просмотр журнала ELMAH

Просмотр журнала ELMAH прост, но сначала создается необработанное исключение, которое будет записано в журнал ELMAH.

1. Нажмите клавиши **CTRL + F5** , чтобы запустить пример приложения Wingtip Toys.
2. Чтобы записать необработанное исключение в журнал ELMAH, перейдите в браузере по следующему URL-адресу (по номеру порта):  
    `https://localhost:44300/NoPage.aspx` отобразится страница ошибки.
3. Чтобы отобразить журнал ELMAH, перейдите в браузере по следующему URL-адресу (по номеру порта):  
    `https://localhost:44300/elmah.axd`
