# <a name="using-visual-studio-2013-to-create-a-basic-aspnet-45-web-forms-page"></a>С помощью Visual Studio 2013 для создания основных ASP.NET 4.5 страницу Web Forms

==== по [Erik Reitan](https://github.com/Erikre)

[!INCLUDE[](~/includes/rp.md)]

В этом пошаговом руководстве предоставляются общие сведения о среде веб-разработки в [Microsoft Visual Studio 2013](https://www.microsoft.com/visualstudio/11/downloads#vs) и в [Microsoft Visual Studio Express 2013 для Web](https://www.microsoft.com/visualstudio/11/downloads#express-web). В этом пошаговом руководстве описывается создание простой страницы веб-форм ASP.NET, а также иллюстрирует основные приемы создания новой страницы, добавление элементов управления и написания кода.

В данном пошаговом руководстве представлены следующие задачи.

- Создание проекта приложения веб-форм файл системы.
- Знакомство с Visual Studio.
- Создание страницы ASP.NET.
- Добавление элементов управления.
- Добавление обработчиков событий.
- Запуск и тестирование страницу из Visual Studio.

## <a name="prerequisites"></a>Предварительные требования

Для выполнения данного пошагового руководства требуется:

- [Microsoft Visual Studio 2013](https://www.microsoft.com/visualstudio/11/downloads#vs) или [Microsoft Visual Studio Express 2013 для Web](https://www.microsoft.com/visualstudio/11/downloads#express-web). Платформа .NET Framework устанавливается автоматически. 

    > [!NOTE] 
    > 
    > Microsoft Visual Studio 2013 и Microsoft Visual Studio Express 2013 для Web будет часто обращаться как к Visual Studio данной серии учебных курсов.  
    >   
    > При использовании Visual Studio, в этом пошаговом руководстве предполагается, что выбрана **веб-разработки** набор параметров при первом запуске Visual Studio. Дополнительные сведения см. в разделе [Как Выберите параметры среды веб-разработки](https://msdn.microsoft.com/library/ff521558.aspx).


## <a name="creating-a-web-application-project-and-a-page"></a>Создание проекта веб-приложения и страницы

<a id="sectionToggle0"></a>

В этой части пошагового руководства будет создать проект веб-приложения и добавьте новую страницу к нему. Кроме того, будет добавить HTML-текст и запустить страницу в браузере.

### <a name="to-create-a-web-application-project"></a>Чтобы создать проект веб-приложения

1. Open Microsoft Visual Studio.
2. В меню **Файл** выберите **Создать проект**.  
    ![Меню "файл"](creating-a-basic-web-forms-page/_static/image1.png)

    Откроется диалоговое окно **Новый проект** .
3. Выберите **шаблоны**  - &gt; **Visual C#**  - &gt; **Web** группы шаблонов в левой части.
4. Выберите **веб-приложение ASP.NET** шаблона в центральном столбце.
5. Присвойте проекту имя ***BasicWebApp*** и нажмите кнопку **ОК** кнопки.   
![Диалоговое окно "новый проект"](creating-a-basic-web-forms-page/_static/image2.png)
6. Затем выберите **веб-форм** шаблона и нажмите кнопку **ОК** кнопку, чтобы создать проект.  
![Диалоговое окно "новый проект ASP.NET"](creating-a-basic-web-forms-page/_static/image3.png)  

    Visual Studio создает новый проект, включающий стандартные функциональные возможности, на основе шаблона веб-форм. Она не только дает возможность *Home.aspx* странице *About.aspx* странице *Contact.aspx* странице, но также включает функции членства, который регистрирует пользователей и сохраняет учетные данные, чтобы они могут войти в веб-сайта. При создании новой страницы по умолчанию Visual Studio отображается страница в **источника** представление, где вы увидите элементы HTML страницы. На следующем рисунке показано, как будет видно **источника** просмотра, если вы создали новую веб-страницу с именем *BasicWebApp.aspx*.

### <a name="creating-a-new-aspnet-web-forms-page"></a>Создание новой страницы ASP.NET Web Forms


Когда вы создаете новое приложение веб-форм с помощью **веб-приложение ASP.NET** шаблон проекта Visual Studio добавляет страницы ASP.NET (страницы веб-форм) с именем *Default.aspx*, а также как несколько файлов и папки. Можно использовать *Default.aspx* страницу как на домашнюю страницу веб-приложения. Тем не менее в этом пошаговом руководстве будет создавать и работать с новой страницы.

### <a name="to-add-a-page-to-the-web-application"></a>Добавление страницы в веб-приложение


1. Закрыть *Default.aspx* страницы. Чтобы сделать это, перейдите на вкладку, в котором отображается имя файла и нажмите кнопку «Закрыть».
2. В **обозревателе решений**, щелкните правой кнопкой мыши имя веб-приложения (в этом руководстве, имя приложения — **BasicWebSite**), а затем нажмите кнопку **добавить**  - &gt; **Новый элемент**.   
Откроется диалоговое окно **Добавление нового элемента**.
3. Выберите **Visual C#**  - &gt; **Web** группы шаблонов в левой части. Выберите **веб-формы** из середины списка и назовите его *FirstWebPage.aspx*.   
    ![Добавить новый элемент-диалоговое окно](creating-a-basic-web-forms-page/_static/image6.png)
4. Нажмите кнопку **добавить** для добавления в проект веб-страницы.  
Visual Studio создаст новую страницу и открывает его.


### <a name="adding-html-to-the-page"></a>Добавление на страницу HTML


В этой части пошагового руководства вы добавите статический текст на страницу.

### <a name="to-add-text-to-the-page"></a>Добавление текста на страницу


1. В нижней части окна документа, щелкните **разработки** для перехода к **разработки** представления.

    Режим конструктора отображает текущую страницу в точного отображения. На этом этапе у вас нет любой текст или элементы управления страницы, чтобы страница пуста, за исключением пунктирной линией, описывающий прямоугольник. Представляет этот прямоугольник **div** элемента на странице.
2. Щелкните внутри прямоугольника, представленной пунктирной линией.
3. Тип **Добро пожаловать в Visual Web Developer** и нажмите клавишу **ввод** дважды.

    На следующем рисунке показан текст, введенный в **разработки** представления.

    ![Текст приветствия в режиме конструктора](creating-a-basic-web-forms-page/_static/image7.png "текст приветствия в режиме конструктора")
4. Переключиться в режим **источника** представления.

    Вы увидите HTML в **источника** представление, которое вы создали при вводе в **разработки** представления.  
    ![Веб-страницы со статическим текстом](creating-a-basic-web-forms-page/_static/image8.png)


### <a name="running-the-page"></a>Загрузка страницы


Прежде чем продолжить, добавив элементы управления на страницу, можно запустить его.

### <a name="to-run-the-page"></a>Чтобы запустить эту страницу


1. В **обозревателе решений**, щелкните правой кнопкой мыши *FirstWebPage.aspx* и выберите **задать в качестве начальной страницы**.
2. Нажмите клавишу **CTRL + F5** для запуска страницы.

    В браузере отображается страница. Несмотря на то, что созданная страница имеет расширение имени файла из *.aspx*, он в настоящее время выполняется подобно любой HTML-страницы.

    Для отображения страницы в браузере вы можете щелкнуть правой кнопкой на странице в **обозревателе решений** и выберите **просмотреть в браузере**.
3. Закройте браузер, чтобы остановить веб-приложения.


## <a name="adding-and-programming-controls"></a>Добавление и программирование элементов управления


<a id="sectionToggle1"></a>

Теперь добавим серверных элементов управления на страницу. Серверные элементы управления, такие как кнопки, метки, текстовые поля и другие знакомые элементы управления предоставляют типичные возможности обработки форм для страниц веб-форм. Тем не менее можно программировать элементы управления с помощью кода, выполняемого на сервере, а не клиента.

Вы добавите [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) элемента управления, [TextBox](https://msdn.microsoft.com/library/system.web.ui.webcontrols.textbox.aspx) элемента управления и [метки](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) на страницу элемент управления и писать код для обработки [щелкните](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.click.aspx) событий для [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) элемента управления.

### <a name="to-add-controls-to-the-page"></a>Для добавления элементов управления на страницу


1. Нажмите кнопку **разработки** для перехода к **разработки** представления.
2. Поместите курсор в конце **Добро пожаловать в Visual Web Developer** текст и нажмите клавишу **ввод** пять или более раз, чтобы освободить место на диске в **div** поле элемента.
3. В **элементов**, разверните **стандартный** группе, если он еще не развернут.  
Обратите внимание, что может потребоваться развернуть **элементов** окна слева, чтобы просмотреть его.
4. Перетащите [TextBox](https://msdn.microsoft.com/library/system.web.ui.webcontrols.textbox.aspx) управления на страницу и поместите его в середине объекта **div** элемент поле, имеющее **Добро пожаловать в Visual Web Developer** в первой строке.
5. Перетащите [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) управления на страницу и поместите его справа от [TextBox](https://msdn.microsoft.com/library/system.web.ui.webcontrols.textbox.aspx) элемента управления.
6. Перетащите [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) управления на страницу и поместите его на отдельной строке под [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) элемента управления.
7. Поместите точку вставки над [TextBox](https://msdn.microsoft.com/library/system.web.ui.webcontrols.textbox.aspx) управления, а затем введите **введите свое имя:** .

    Этот статический HTML-текст является заголовок для [TextBox](https://msdn.microsoft.com/library/system.web.ui.webcontrols.textbox.aspx) элемента управления. Можно смешивать статические HTML и серверных элементов управления на одной странице. На следующем рисунке показано, как отображаются три элемента управления в **разработки** представления.

    ![Три элемента управления в режиме конструктора](creating-a-basic-web-forms-page/_static/image9.png "три элемента управления в режиме конструктора")


### <a name="setting-control-properties"></a>Установка свойств элемента управления


Visual Studio предлагает различные способы установки свойств элементов управления на странице. В этой части пошагового руководства будут заданы свойства в обоих **разработки** представление и **источника** представления.

### <a name="to-set-control-properties"></a>Задание свойств элемента управления


1. Во-первых, отображения **свойства** windows путем выбора из **представление** меню -&gt; **Other Windows**  - &gt; **Окно свойств**. Можно также выбрать **F4** для отображения **свойства** окна.
2. Выберите [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) элемента управления, а затем в **свойства** окна, установите для параметра **текст** для **отображаемое имя**. Введенный текст отображается на кнопке в конструкторе, как показано на следующем рисунке.

    ![Задать текст кнопки](creating-a-basic-web-forms-page/_static/image10.png "кнопку задать текст")
3. Переключиться в режим **источника** представления.

    **Источник** представление отображает HTML для страницы, включая элементы, созданные в Visual Studio для серверных элементов управления. Объявления элементов управления схожий с HTML-синтаксис **asp:** и включает атрибут **runat =&quot;server&quot;**.

    Свойства элемента управления объявляются как атрибуты. Например, если выбрано [текст](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.text.aspx) свойство для [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) управления, на шаге 1, фактически устанавливается **текст** атрибут в разметке элемента управления.

    > [!NOTE] 
    > 
    > Все элементы управления находятся внутри **формы** элемент, который также имеет атрибут **runat =&quot;server&quot;**. **Runat =&quot;server&quot;**  атрибут и **asp:** префикс элемента управления элементами управления, чтобы они обрабатываются платформой ASP.NET на сервере при запуске страницы. Код вне элементов **&lt;форме runat =&quot;server&quot; &gt;** и **&lt;скрипт runat =&quot;server&quot; &gt;** передается без изменений в браузер, поэтому код ASP.NET должен быть внутри элемента, открывающий тег содержит **runat =&quot;server&quot;**  атрибута.
4. Далее, добавьте дополнительные свойства [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) элемента управления. Поместите точку вставки непосредственно после **asp: Label** в **&lt;asp: Label&gt;** тег, а затем нажмите клавишу **пробел**.

    С раскрывающимся списком отображается список доступных свойств, можно задать для [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) элемента управления. Эта функция называется **IntelliSense**, поможет вам в **источника** представление с синтаксисом серверных элементов управления, HTML-элементов и других элементов на странице. На следующем рисунке показано **IntelliSense** стрелку раскрывающегося списка для [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) элемента управления.

    ![Атрибуты IntelliSense](creating-a-basic-web-forms-page/_static/image11.png "атрибуты IntelliSense")
5. Выберите **ForeColor** и введите знак равенства.

    IntelliSense отображает список цветов.

    > [!NOTE] 
    > 
    > Можно отобразить **IntelliSense** стрелку раскрывающегося списка в любое время, нажав клавишу **CTRL + J** при просмотре кода.
6. Выберите цвет для **[метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx)** текст элемента управления. Убедитесь, что вы выбрали цвет, который является темной, для чтения на белом фоне.

    **ForeColor** атрибут выполнен в цвет, который вы выбрали, включая знак закрывающей кавычки.


### <a name="programming-the-button-control"></a>Программирование элемента управления Button


В этом пошаговом руководстве вы напишете код, который считывает имя, что пользователь вводит в текстовое поле и затем отображает имя в [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) элемента управления.

### <a name="add-a-default-button-event-handler"></a>Добавление обработчика событий кнопки по умолчанию


1. Переключиться в режим **разработки** представления.
2. Дважды щелкните [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) элемента управления.

    По умолчанию Visual Studio переключается в файл с выделенным кодом и создает каркас обработчика событий для [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) событие элемента управления по умолчанию, [щелкните](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.click.aspx) событий. Файл с выделенным кодом отделяет разметки пользовательского интерфейса (например, HTML) из кода сервера (например, C#).   
   Курсор располагается добавлен код для этого обработчика событий.

    > [!NOTE]

### <a name="running-the-page"></a>Загрузка страницы


Теперь можно протестировать серверных элементов управления на странице.

### <a name="to-run-the-page"></a>Чтобы запустить эту страницу


1. Нажмите клавишу **CTRL + F5** для запуска страницы в браузере. Если возникает ошибка, проверьте описанные выше действия.
2. Введите имя в текстовое поле и нажмите кнопку **отображаемое имя** кнопки.

    Введенное имя отображается в [метка](https://msdn.microsoft.com/library/system.web.ui.webcontrols.label.aspx) элемента управления. Обратите внимание на то, что при нажатии кнопки, страница передается на веб-сервер. Затем ASP.NET заново создает страницу, выполняет код (в этом случае [кнопку](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.aspx) элемента управления [щелкните](https://msdn.microsoft.com/library/system.web.ui.webcontrols.button.click.aspx) выполняется обработчик события) и затем отправляет новую страницу в браузер. Если смотреть строке состояния в браузере, вы увидите страницы что круговом пути на веб-сервер при каждом нажатии кнопки.
3. В браузере, просмотрите исходный код страницы, выполняется, щелкнув правой кнопкой мыши на странице и выбрав **Просмотр исходного кода**.

    В исходный код страницы вы увидите HTML без кода сервера. В частности, вы не видите **&lt;asp:&gt;** элементы, которые вы работали с **источника** представления. При запуске страницы ASP.NET обрабатывает серверные элементы управления и отображает HTML-элементы страницы, предназначенные для задач, которые представляют элемент управления. Например **&lt;asp: Button&gt;** элемент управления отрисовывается как HTML **&lt;тип входного =&quot;отправить&quot; &gt;** элемент.
4. Закройте браузер.
