# <a name="using-page-inspector-for-visual-studio-2012-in-aspnet-web-forms"></a>Использование инспектора страниц для Visual Studio 2012 в веб-формах ASP.NET

Тим Ammann

> Инспектор страниц для Visual Studio 2012 — это средство веб-разработки с интегрированным браузером. Выберите любой элемент в встроенного браузера и инспектор страниц мгновенно выделения источника этого элемента и CSS. Можно просмотреть любой страницы в приложении, быстро находить источники визуализированной разметке и использовать средства браузера непосредственно в среде Visual Studio.

> Этом руководстве показано, как включить режим инспектирования и быстро найти и отредактировать правила CSS и текста в веб-проект. В этом руководстве используется проект веб-форм приложения, но можно также использовать инспектор страниц для проектов веб-сайтов и [MVC](https://go.microsoft.com/?linkid=9802002) приложений.

## <a name="prerequisites"></a>Предварительные требования

- [Visual Studio 2012](https://www.microsoft.com/visualstudio/11) или [Visual Studio Express 2012 для Web](https://www.microsoft.com/visualstudio/11/downloads#express-web).

> [!NOTE]
> Чтобы получить последнюю версию инспектор страниц, используйте [установщика веб-платформы](https://go.microsoft.com/fwlink/?LinkId=255386) для установки пакета Azure SDK для .NET 2.0.

Инспектор страниц входит в состав инструменты разработчика Microsoft Web. Последняя версия — 1.3. Чтобы проверить, какая версия у, запустите Visual Studio и выберите **о Microsoft Visual Studio** из **помочь** меню.

![Новое приложение Web Forms](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image1.png)

Нажмите кнопку **ОК**.

Приложение откроется в **источника** представления.

![Новые приложения веб-форм в представлении источника](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image2.png)

Теперь, когда у вас есть приложение для работы с, инспектор страниц можно использовать для проверки и изменения его.

На нижней панели отображаются страницы в разметке HTML в левой части, а некоторые вкладки в правой части, которые позволяют проверять различные аспекты страницы. На нижней панели аналогичен [средства разработчика F12](https://msdn.microsoft.com/ie/aa740478) в Internet Explorer. <a id="_4_inspection_mode"></a>
## <a name="enable-inspection-mode"></a>Включить режим проверки

Далее вы увидите, как работает режим инспектирования инспектора страниц. В окне инспектора страниц щелкните **инспектировать** кнопки.

![Проверить элемент](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image7.png)

Чтобы просмотреть режим инспектирования в действии, в различные части страницы в окне браузера инспектора страниц наведите указатель мыши. После этого указатель мыши примет знака плюс, и выделяется под элементом:

![При наведении div.content оболочки](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image8.png)

При перемещении указателя мыши, обратите внимание, что

- Содержимое в **источника** просмотра изменяется для отображения разметки, соответствующий выбранному элементу на странице. Соответствующая разметка выделяется. Если источник данных находится в другой файл, этот файл открывается в представлении источника соответствующие выделенным разметкой.

- Разметка, отображаемая в **HTML** вкладку в инспекторе страниц также изменяется в соответствии с выбранного элемента на странице. В **HTML** вкладке описана соответствующая разметка.

- **Стили** вкладке отображаются правила CSS, соответствующие к текущему выделению.

<a id="_5_using_page"></a>

## <a name="use-page-inspector-to-make-changes-to-markup"></a>Использование инспектора страниц вносить изменения в разметку

Теперь вы увидите, как инспектор страниц можно использовать для поиска и внести изменения в разметку или текст, расположение которой может не всегда самоочевиден.

Перевести инспектор страниц в режиме инспектирования, а затем прокрутите до нижней части домашней страницы.

Как только вы введете области нижнего колонтитула, инспектор страниц открывает *Site.Master* файл макета в **источника** представления в временную вкладку справа от другой вкладки и выделяет часть главной странице, выбраны. Это показано, как инспектор страниц можно найти и отображения содержимого на страницу, которая может действительно получены от той, которую была открыта другой файл.

![Основные особенности нижний колонтитул в режиме инспектирования](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image9.png)

В окне браузера инспектора страниц наведите указатель мыши на строку с авторских прав <a id="a"> </a>Обратите внимание, что.

В *Site.Master* страницы, соответствующая строка будет выделена.

![Строка нижнего колонтитула об авторских правах, выделяемая подсветкой](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image10.png)

Добавьте какой-нибудь текст в конец строки в *Site.Master* файл.

&lt;p&gt;&amp;copy; &lt;%: DateTime.Now.Year %&gt; — My ASP.NET Application Rocks!&lt; /p&gt;

Теперь нажмите клавиши Ctrl + Alt + Ввод или щелкните панель обновления, чтобы просмотреть результаты в окне браузера инспектора страниц.

![My ASP.NET Application Rocks!](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image11.png)

Вы наверняка считали, что нижний колонтитул был на *Default.aspx* страницы, но оно оказалось в страница с эталонной разметкой, и инспектор страниц нашли для вас.

## <a name="preview-css-changes-in-the-styles-window"></a>Просмотр изменений CSS в окне "Стили"

Далее, вы увидите, как можно использовать инспектор страниц **стили** окно, чтобы просмотреть изменения в CSS.

Нажмите кнопку **инспектировать** кнопку, чтобы перевести инспектор страниц в режиме инспектирования.

В окне браузера инспектора страниц наведите указатель мыши на раздел «Домашняя страница» до **div.content оболочки** появится метка.

![Наведение на элементы](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image15.png)

Щелкните в разделе div.content оболочку, а затем наведите указатель мыши на **стили** окна. В группе селектора класса .featured .content оболочки снимите и установите флажок для свойства цвет фона.

![Цвет фона очистить](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image16.png)

Обратите внимание на то, как изменение выполняет предварительный просмотр сразу же в окне браузера инспектора страниц.

Снова установите флажок, а затем дважды щелкните значение свойства и измените его на `red`. Изменение немедленно показано:

![Красного цвета фона](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image17.png)

**Стили** облегчает тестирование и предварительный просмотр CSS изменения, прежде чем зафиксировать изменения стиля делает окно листа сам.

<a id="css_auto_sync"></a>
## <a name="css-auto-sync"></a>Автосинхронизация CSS

> [!NOTE]
> Эта функция требует версии 1.3 инспектора страниц.

Автосинхронизация CSS позволяет напрямую изменить CSS-файл и увидеть изменения прямо в браузере инспектора страниц.

Нажмите кнопку **инспектировать** чтобы перевести инспектор страниц в режиме инспектирования.

В браузере инспектора страниц наведите указатель мыши на раздел «Домашняя страница» до **div.content оболочки** появится метка. Щелкните один раз, чтобы выбрать этот элемент.

**Стили** окне показаны все правила CSS для данного элемента. Прокрутите вниз до селектор класса поиска .featured .content оболочки. Щелкните «.featured .content оболочки». Инспектор страниц открывает CSS-файл, который определяет этот стиль (Site.css) и выделяет соответствующий стиль CSS.

![CSS-файл](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image18.png)

Теперь измените значение `background-color` «красный». Изменение немедленно отображается в браузере инспектора страниц.

![Браузере инспектора страниц](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image19.png)

<a id="_using_the_visual"></a>

### <a name="using-the-visual-studio-2012-css-color-picker"></a>С помощью Visual Studio 2012 палитра цветов CSS

Редактор CSS в Visual Studio 2012 содержит палитру цветов, позволяет легко выбрать и вставить цвета. Она имеет простой цветовую шкалу и выбора «pop-down», которое предлагает возможности более точного управления.

Палитра содержит стандартную палитру цветов, поддерживает стандартные имена цветов, хэш-кодов, цвета RGB, RGBA, HSL и HSLA и ведет список цветов, которые наиболее недавно использовавшегося в документе.

В строке, где был в свойстве цвета фона введите «bc» и нажмите стрелку вниз один раз.

При вводе первый символ каждого слова в свойства, разделенных дефисом, такого как «background-color» технология IntelliSense отфильтровывает список для отображения только свойств, которые соответствуют:

![Фильтрация IntelliSense значения](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image24.png)

Теперь введите двоеточие. При этом вставляется имя свойства полный цвет фона. Тип **#** или **rgb (**, и появляется палитра цветов:

![Палитра цветов CSS](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image25.png)

Чтобы увидеть, как работает средство выбора цветов, щелкните его цвета с указателем, или нажмите клавишу со стрелкой вниз и затем с помощью клавиш со стрелками влево и вправо для обхода цвета. При посещении цвет, предварительном просмотре соответствующее значение для свойства цвет фона:

![значение свойства цвет фона предварительного просмотра](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image26.png)

На этом этапе можно нажать ВВОД, чтобы выбрать значение, а затем точку с запятой (;) для завершения записи CSS. Сейчас перейдите к следующему разделу, чтобы можно было увидеть, как работает средство выбора цвета pop вниз.

#### <a name="using-the-color-picker-pop-down"></a>С помощью средства выбора цвета Pop вниз

При цветовой полосы не имеет необходимого цвета, который вы ищете, можно использовать палитра pop вниз.

Чтобы открыть его, щелкните значок шеврона в правом конце цветовой полосы, или один или два раза нажмите клавишу со стрелкой вниз на клавиатуре.

![Средство выбора цвета CSS, Pop вниз](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image27.png)

Выберите цвет из вертикальная черта справа. Это показывает градиента цвета в главном окне. Выбрать цвет непосредственно из вертикальной чертой, нажав клавишу ВВОД или щелкните любую точку в главном окне для выбора с большей точностью.

Есть ли цвет на экране компьютера, который вы хотите использовать (он не должен быть в пользовательском интерфейсе Visual Studio), его значение можно захватить, используя инструмент "Пипетка" в правом нижнем углу.

Также можно изменить непрозрачность цвета, переместив ползунок в нижней части палитры цветов. Способ изменения значения RGBA значения цветов, так как формат RGBA может представлять непрозрачности.

После выбора цвета, нажмите клавишу ВВОД, а затем введите точку с запятой, чтобы завершить ввод цвет фона в *Site.css* файл.

### <a name="the-page-inspector-update-bar"></a>Панель обновления инспектора страницы

Инспектор страниц немедленно обнаруживает изменение *Site.css* файл (или любой другой файл в приложении) и отображает предупреждение в панель обновления.

![Панель обновления](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image28.png)

Чтобы сохранить все файлы и обновите содержимое браузера инспектора страниц, нажмите клавиши Ctrl + Alt + Ввод или щелкните панель обновления. Изменение цвета выделения откроется в браузере:

![Изменить цвет выделения](using-page-inspector-in-a-visual-studio-11-beta-web-forms-project/_static/image29.png)

<a id="_using_page_inspector_1"></a>Обратите внимание, что удобно обновление браузера инспектора страниц прямо из среды Visual Studio. Использование инспектора страниц, а не во внешнем браузере позволяет оставаться в редакторе при разработке веб-приложений.
