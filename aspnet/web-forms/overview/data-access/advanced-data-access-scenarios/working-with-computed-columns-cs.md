## <a name="summary"></a><span data-ttu-id="e2389-282">Сводка</span><span class="sxs-lookup"><span data-stu-id="e2389-282">Summary</span></span>

<span data-ttu-id="e2389-283">При определении схемы для таблицы, Microsoft SQL Server позволяет включать вычисляемые столбцы.</span><span class="sxs-lookup"><span data-stu-id="e2389-283">When defining the schema for a table, Microsoft SQL Server allows the inclusion of computed columns.</span></span> <span data-ttu-id="e2389-284">Это те столбцы, значения которого вычисляются по выражению, обычно ссылается на значения из других столбцов в той же записи.</span><span class="sxs-lookup"><span data-stu-id="e2389-284">These are columns whose values are calculated from an expression that usually references the values from other columns in the same record.</span></span> <span data-ttu-id="e2389-285">Так как значения для вычисляемых столбцов основана на выражении, они доступны только для чтения и не может быть присвоено значение в `INSERT` или `UPDATE` инструкции.</span><span class="sxs-lookup"><span data-stu-id="e2389-285">Since the values for computed columns are based on an expression, they are read-only and cannot be assigned a value in an `INSERT` or `UPDATE` statement.</span></span> <span data-ttu-id="e2389-286">Это создает проблемы при использовании вычисляемый столбец в основном запросе адаптера таблицы, которая пытается автоматически создают соответствующие `INSERT`, `UPDATE`, и `DELETE` инструкций.</span><span class="sxs-lookup"><span data-stu-id="e2389-286">This introduces challenges when using a computed column in the main query of a TableAdapter that tries to automatically generate corresponding `INSERT`, `UPDATE`, and `DELETE` statements.</span></span>
