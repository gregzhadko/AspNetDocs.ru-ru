### <a name="asynchronously-reading-and-writing-http-requests-and-responses"></a><span data-ttu-id="14106-177">Асинхронно чтение и написание запросов и ответов HTTP</span><span class="sxs-lookup"><span data-stu-id="14106-177">Asynchronously Reading and Writing HTTP Requests and Responses</span></span>

<span data-ttu-id="14106-178">ASP.NET 4 ввел возможность чтения сущности запроса HTTP в качестве потока с помощью метода *HttpRequest.GetBufferlessInputStream.*</span><span class="sxs-lookup"><span data-stu-id="14106-178">ASP.NET 4 introduced the ability to read an HTTP request entity as a stream using the *HttpRequest.GetBufferlessInputStream* method.</span></span> <span data-ttu-id="14106-179">Этот метод обеспечил потоковое использование доступа к сущности запроса.</span><span class="sxs-lookup"><span data-stu-id="14106-179">This method provided streaming access to the request entity.</span></span> <span data-ttu-id="14106-180">Тем не менее, он выполнил синхронно, который связал поток на время запроса.</span><span class="sxs-lookup"><span data-stu-id="14106-180">However, it executed synchronously, which tied up a thread for the duration of a request.</span></span>

<span data-ttu-id="14106-181">ASP.NET 4.5 поддерживает способность считывать потоки асинхронно на сущности запроса HTTP и способность к асинхронному флешу.</span><span class="sxs-lookup"><span data-stu-id="14106-181">ASP.NET 4.5 supports the ability to read streams asynchronously on an HTTP request entity, and the ability to flush asynchronously.</span></span> <span data-ttu-id="14106-182">ASP.NET 4.5 также дает возможность двабуфера сущность запроса HTTP, которая обеспечивает более легкую интеграцию с обработчиками http-потока, такими как обработчики страниц .aspx и ASP.NET контроллеров MVC.</span><span class="sxs-lookup"><span data-stu-id="14106-182">ASP.NET 4.5 also gives you the ability to double-buffer an HTTP request entity, which provides easier integration with downstream HTTP handlers such as .aspx page handlers and ASP.NET MVC controllers.</span></span>

class="sxs-lookup"><span data-stu-id="14106-189">However, as it reads, *GetBufferedInputStream* also copies the entity bytes into ASP.NET internal buffers so that downstream modules and handlers can
