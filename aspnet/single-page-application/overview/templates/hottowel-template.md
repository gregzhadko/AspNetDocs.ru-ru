## <a name="application-structure"></a>Структура приложений

"Горячий" SPA выкинуть предоставляет папки приложения, который содержит файлы JavaScript и HTML, которые определяют приложение.

Внутри папки приложения:

- durandal
- службы
- ViewModels
- представления

Папка приложения содержит коллекцию модулей. Эти модули инкапсулируют функциональность и объявлять зависимости на другие модули. Папка views содержит HTML-код для приложения и в папку viewmodels содержит логику представления для представлений (общий шаблон MVVM). Папка служб идеально подходит для размещения любые общие службы, которые могут потребоваться приложению, такие как получение данных HTTP или взаимодействия локального хранилища. Довольно часто несколько viewmodels для повторного использования кода из модулей службы.

## <a name="how-do-i-build-on-hot-towel"></a>Как создать на "Горячий" выкинуть?

Просто приступите к добавлению кода!

1. Добавьте свой код на стороне сервера, предпочтительно Entity Framework и веб-API (которые предстают Breeze.js)
2. Добавление представлений для `App/views` папки
3. Добавление viewmodels `App/viewmodels` папки
4. Добавить HTML и Knockout привязки данных в новые представления
5. Обновление маршрутов навигации в `shell.js`
