## <a name="server-system-requirements"></a>Системные требования для сервера

Компонент сервера SignalR могут размещаться на различных конфигураций сервера. В этом разделе описываются поддерживаемые версии операционных систем, .NET framework, Internet Information Server и других компонентов.

### <a name="supported-server-operating-systems"></a>Поддерживаемые серверные операционные системы

Компонент сервера SignalR могут размещаться в следующих операционных систем сервера или клиента. Обратите внимание, что для SignalR использование WebSockets, Windows Server 2012, Windows Server 2016 или Windows 8 является обязательным (WebSocket можно использовать на Windows Azure Web Sites, поскольку имеет значение узла версии платформы .NET framework 4.5 и включения веб-сокеты на сайте Страница конфигурации).

- Windows Server 2016
- Windows Server 2012
- Windows Server 2008 r2
- Windows 10
- Windows 8
- Windows 7
- Microsoft Azure

### <a name="supported-server-net-framework-version"></a>Поддерживаемые серверные версии платформы .NET Framework

SignalR 2 поддерживается только в .NET Framework 4.5. См. в разделе [рекомендованные обновления](#updates) раздел для обновлений, которые повышают надежность, совместимости, стабильности и производительности.

### <a name="supported-server-iis-versions"></a>Поддерживаемых версиях IIS

Если SignalR размещается в службах IIS, поддерживаются следующие версии. Обратите внимание, что при использовании клиентской операционной системы, такие как для разработки приложений (Windows 8 или Windows 7), полные версии IIS или Cassini не следует, поскольку будет ограничение в 10 одновременных подключений накладываются, которых истекает очень быстро с момента подключения несохраняемым, часто повторно установить и они не удаляются, сразу же после больше не используются. IIS Express можно использовать в клиентских операционных системах.

Также Обратите внимание, что для SignalR для использования веб-сокет, должен использоваться IIS 8 или IIS 8 Express, сервер необходимо использовать Windows 8, Windows Server 2012 или более поздней версии, и требуется включить WebSocket в IIS. Сведения о том, как включить WebSocket в IIS, см. в разделе [поддержка протокола WebSocket в IIS 8.0](https://www.iis.net/learn/get-started/whats-new-in-iis-8/iis-80-websocket-protocol-support).

- IIS 8 или IIS 8 Express.
- IIS 7 и 7.5. Поддержка [URL-адреса без расширений приложения](https://support.microsoft.com/kb/980368) является обязательным.
- IIS должен работать в режиме интеграции с; классический режим не поддерживается. Могли возникнуть задержки сообщений до 30 секунд, если IIS выполняется в классическом режиме с помощью транспорта Server-Sent события.
- Ведущее приложение должно выполняться в режиме полного доверия.

## <a name="client-system-requirements"></a>Системные требования для клиента

SignalR может использоваться в различных клиентских платформ. В этом разделе описываются требования к системе для использования SignalR в веб-браузеров, настольных приложений Windows, приложений Silverlight и мобильных устройств.

### <a name="web-browsers"></a>Веб-браузеры

SignalR может использоваться в различных браузерах, но обычно поддерживаются только две последние версии.

Приложения, использующие SignalR в браузерах необходимо использовать версию jQuery 1.6.4 или основных более поздних версий (например, 1.7.2, 1.8.2 или 1.9.1).

SignalR может использоваться в следующих браузерах:

- Microsoft Internet Explorer версии 8, 9, 10 и 11. Современный, рабочий стол и мобильные версии поддерживаются.
- Mozilla Firefox: текущая версия - 1, версии, Mac и Windows.
- Google Chrome: текущая версия - 1, версии, Mac и Windows.
- Safari: текущая версия - 1, версии, Mac и iOS.
- Opera: текущая версия — 1, только для Windows.
- Браузер Android

Помимо необходимости обозревателей, различных транспортов, которые использует SignalR имеют свои собственные требования. Поддерживаются следующие транспорты следующие конфигурации:

<a id="browser"></a>

**Требования к транспорта обозревателя в Интернете**

| Transport | Internet Explorer | Chrome (Windows или iOS) | Firefox | Safari (OSX или iOS) | Android |
| --- | --- | --- | --- | --- | --- |
| Протокол WebSocket | 10+ | Текущая - 1 | Текущая - 1 | Текущая - 1 | Н/Д |
| Сервер отправил события | Н/Д | Текущая - 1 | Текущая - 1 | Текущая - 1 | Н/Д |
| ForeverFrame | 8+ | Н/Д | Н/Д | Н/Д | 4.1 |
| Долгие опросы | 8+ | Текущая - 1 | Текущая - 1 | Текущая - 1 | 4.1 |

\*: 6 + необходимые для обеспечения полной функциональности.

#### <a name="unsupported-browsers"></a>Неподдерживаемые браузеры

Хотя SignalR *может* работали без особых проблем, в предыдущих версиях обозревателя, мы активно не тестируйте SignalR в них и обычно не исправить ошибки, которые могут появиться в них.

### <a name="windows-desktop-and-silverlight-applications"></a>Рабочий стол Windows и приложения Silverlight

Помимо выполнения в веб-браузер, SignalR могут размещаться в автономный клиент Windows или приложений Silverlight. Приложения рабочего стола Windows и Silverlight SignalR имеют следующие требования к системе.

- Приложений с помощью .NET 4, поддерживаются в Windows XP с пакетом обновления 3 или более поздней версии.
- Приложений с помощью .NET Framework 4.5, поддерживаются в Windows Vista или более поздней версии.

Помимо операционной системы и требования к .NET framework транспорты, доступных для SignalR имеют свои собственные требования. Поддерживаются следующие транспорты следующие конфигурации:

**Windows Desktop и требования к Silverlight транспорта**

| Transport | Приложения .NET | Silverlight |
| --- | --- | --- |
| Веб-сокеты | Windows 8 + и .NET 4.5 + | Н/Д |
| Навсегда кадра | Н/Д | Н/Д |
| Сервер отправил события | .NET 4+ | 5+ |
| Долгие опросы | .NET 4+ | 5+ |
