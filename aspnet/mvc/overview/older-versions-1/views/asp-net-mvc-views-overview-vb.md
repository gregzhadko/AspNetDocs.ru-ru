## <a name="understanding-views"></a>Общие сведения о представлениях

В отличие от ASP.NET или страницы ASP ASP.NET MVC не включает все, что соответствует непосредственно на страницу. В приложении ASP.NET MVC не страницы на диск, который соответствует пути в URL-адрес, который вы вводите в адресной строке браузера. -Это ближайший страницы в приложении ASP.NET MVC, то вызывается *представление*.

В приложении ASP.NET MVC входящий запрос браузера, сопоставляются действий контроллера. Действие контроллера может вернуть представление. Тем не менее действия контроллера могут применяться другие действия, такие как перенаправление для другого действия контроллера.

В листинге 1 содержит простой контроллер с именем HomeController. HomeController предоставляет два действия контроллера с именем Index() и Details().

**В листинге 1 - HomeController.vb**

[!code-vb[Main](asp-net-mvc-views-overview-vb/samples/sample1.vb)]

Первое действие, действие Index(), можно вызвать, введя следующий URL-адрес в адресной строке браузера:

/ Home/Index

Второе действие, действие Details(), можно вызвать, введя этот адрес в адресную строку браузера:

/ Home/сведения

Index() действие возвращает представление. Большинство операций, создаваемых вернет представления. Тем не менее действия могут возвращать другие типы результатов действий. Например действие Details() возвращает RedirectToActionResult, который перенаправляет входящий запрос Index() действие.

Действие Index() содержит следующие одной строки кода:

View()

Эта строка кода возвращает представление, которое должен быть расположен по следующему пути веб-сервера:

\Views\Home\Index.aspx

Путь к представлению выводится из имени контроллера и имя действия контроллера.

При желании можно явно задать, какие представлении. Следующая строка кода возвращает представление с именем Fred:

Представление (Fred)

Когда выполняется эта строка кода, представление возвращается из следующий путь:

\Views\Home\Fred.aspx

> [!NOTE] 
> 
> Если вы планируете создавать модульные тесты для своего приложения ASP.NET MVC является хорошей идеей явное имена представлений. Таким образом, можно создать модульный тест, чтобы убедиться, что ожидаемый Просмотр вернула действия контроллера.
