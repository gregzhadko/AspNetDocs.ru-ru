<span data-ttu-id="5a033-157">В приведенном выше сценарии наш класс DinnerFormViewModel непосредственно предоставляет объект модели Dinner как свойство, а также вспомогательное свойство модели SelectList.</span><span class="sxs-lookup"><span data-stu-id="5a033-157">In the scenario above, our DinnerFormViewModel class directly exposes the Dinner model object as a property, along with a supporting SelectList model property.</span></span> <span data-ttu-id="5a033-158">Этот подход прекрасно подходит для сценариев, где HTML-образный универс, который мы хотим создать в шаблоне представления, относительно близко соответствует нашим объектам модели домена.</span><span class="sxs-lookup"><span data-stu-id="5a033-158">This approach works fine for scenarios where the HTML UI we want to create within our view template corresponds relatively closely to our domain model objects.</span></span>

<span data-ttu-id="5a033-159">Для сценариев, в которых это не так, одним из вариантов, который можно использовать, является создание класса ViewModel, объектная модель которого более оптимизирована для потребления представлением и которая может выглядеть совершенно иначе, чем базовый объект модели домена.</span><span class="sxs-lookup"><span data-stu-id="5a033-159">For scenarios where this isn't the case, one option that you can use is to create a custom-shaped ViewModel class whose object model is more optimized for consumption by the view – and which might look completely different from the underlying domain model object.</span></span> <span data-ttu-id="5a033-160">Например, он потенциально может подвергать различные имена свойств и/или агрегированные свойства, собранные с нескольких объектов модели.</span><span class="sxs-lookup"><span data-stu-id="5a033-160">For example, it could potentially expose different property names and/or aggregate properties collected from multiple model objects.</span></span>

<span data-ttu-id="5a033-161">Классы ViewModel в форме пользовательской формы могут использоваться как для передачи данных от контроллеров к представлениям для визуализации, так и для обработки данных формы, размещенных обратно в метод действия контроллера.</span><span class="sxs-lookup"><span data-stu-id="5a033-161">Custom-shaped ViewModel classes can be used both to pass data from controllers to views to render, as well as to help handle form data posted back to a controller's action method.</span></span> <span data-ttu-id="5a033-162">Для этого более позднего сценария метод действия может обновить объект ViewModel с размещенными в форме данными, а затем использовать экземпляр ViewModel для картирования или извлечения фактического объекта модели домена.</span><span class="sxs-lookup"><span data-stu-id="5a033-162">For this later scenario, you might have the action method update a ViewModel object with the form-posted data, and then use the ViewModel instance to map or retrieve an actual domain model object.</span></span>

<span data-ttu-id="5a033-163">Классы ViewModel, сформированные на индивидуальной основе, могут обеспечить большую гибкость и исследовать их в любое время, когда вы находите код рендеринга в шаблонах представления или код размещения формы внутри ваших методов действий, начинают усложняться.</span><span class="sxs-lookup"><span data-stu-id="5a033-163">Custom-shaped ViewModel classes can provide a great deal of flexibility, and are something to investigate any time you find the rendering code within your view templates or the form-posting code inside your action methods starting to get too complicated.</span></span> <span data-ttu-id="5a033-164">Это часто является признаком того, что ваши модели доменов не соответствуют пользовательскому пользовательскому элементу, который вы создаете, и что промежуточный класс ViewModel в форме пользовательской формы может помочь.</span><span class="sxs-lookup"><span data-stu-id="5a033-164">This is often a sign that your domain models don't cleanly correspond to the UI you are generating, and that an intermediate custom-shaped ViewModel class can help.</span></span>
