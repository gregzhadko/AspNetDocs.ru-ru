Можно просматривать любого представления MVC, быстро находить источники визуализированной разметке и использовать средства браузера непосредственно в среде Visual Studio.

## <a name="prerequisites"></a>Предварительные требования

- [Visual Studio 2012](https://www.microsoft.com/visualstudio/11) или [Visual Studio Express 2012 для Web](https://www.microsoft.com/visualstudio/11/downloads#express-web).

> Чтобы получить последнюю версию инспектор страниц, используйте [установщика веб-платформы](https://go.microsoft.com/fwlink/?LinkId=255386) установке Windows Azure SDK для .NET 2.0.


Инспектор страниц входит в состав инструменты разработчика Microsoft Web. Последняя версия — 1.3. Чтобы проверить, какая версия у, запустите Visual Studio и выберите **о Microsoft Visual Studio** из **помочь** меню.

## <a name="create-a-web-application"></a>Создание веб-приложения

Во-первых создайте веб-приложения, который будет использоваться инспектор страниц с. В Visual Studio, выберите **файл** &gt; **новый проект**. Слева разверните **Visual C#** выберите **Web**, а затем выберите **веб-приложение ASP.NET MVC 4**.

![Новое приложение ASP.NET MVC](using-page-inspector-in-aspnet-mvc/_static/image2.png)

Нажмите кнопку **ОК**.

В **создания проекта ASP.NET MVC 4** выберите **веб-приложение**. Оставьте **Razor** как обработчик представлений по умолчанию.

![Новый проект ASP.NET MVC — веб-приложение](using-page-inspector-in-aspnet-mvc/_static/image4.png)

Приложение откроется в **источника** представления.

![Новое приложение ASP.NET MVC в представлении источника](using-page-inspector-in-aspnet-mvc/_static/image6.png)

Теперь, когда у вас есть приложение для работы с, инспектор страниц можно использовать для проверки и изменения его.

<a id="_starting_page_inspector"></a><a id="_3_using_page"></a>

## <a name="use-page-inspector-to-browse-to-a-view"></a>Использование инспектора страниц для обзора в представление

В Visual Studio 2012, щелкнуть правой кнопкой мыши любого представления в проект и выберите **просмотреть в инспекторе страниц**, инспектор страниц будет и выяснить, маршрут на страницу.

В **обозревателе решений**, разверните **представления** папку и затем **Главная** папки. Файл Index.cshtml щелкните правой кнопкой мыши и выберите **просмотреть в инспекторе страниц**.

![Просмотр в инспекторе страниц Index.cshtml](using-page-inspector-in-aspnet-mvc/_static/image8.png)

По умолчанию инспектор страниц расположенной в левой части среды Visual Studio как окно. При желании можно закрепить его в другом месте или открепить окно. См. практическое руководство по [ размещать и закреплять окна](https://msdn.microsoft.com/library/z4y0hsax.aspx).

В верхней области окна инспектора страниц показываются текущей страницы в окне браузера. На нижней панели отображаются страницы в разметке HTML, а также некоторые вкладки, которые позволяют проверять различные аспекты страницы. На нижней панели аналогичен [средства разработчика F12](https://msdn.microsoft.com/ie/aa740478) в Internet Explorer.

![Приложения ASP.NET MVC в инспекторе страниц](using-page-inspector-in-aspnet-mvc/_static/image10.png)

В этом руководстве вы воспользуетесь **HTML** и **стили** вкладки для быстрого перехода и внесения изменений в приложение.

<a id="_examining_(&quot;decomposing&quot;)_the"></a><a id="_inspection_mode_and"></a><a id="_4_inspection_mode"></a>

## <a name="enableinspection-mode"></a>Режим EnableInspection

Чтобы перевести инспектор страниц в режиме инспектирования, нажмите кнопку **инспектировать** кнопки. В режиме инспектирования Если навести указатель мыши на любую часть отображенной странице соответствующая исходная разметка или код будет выделен.

![Переключиться в режим проверки](using-page-inspector-in-aspnet-mvc/_static/image12.png)

Теперь наведите указатель мыши на различные части страницы в инспекторе страниц. После этого указатель мыши примет знака плюс, и выделяется под элементом:

![При наведении div.content оболочки](using-page-inspector-in-aspnet-mvc/_static/image14.png)

При перемещении указателя мыши, Visual Studio выделяет соответствующий синтаксис Razor в исходном файле. Если HTML-элемента берется из другого файла источника, Visual Studio автоматически открывает файл.

В инспекторе страниц **HTML** вкладке отображается HTML-код, который был создан из синтаксиса Razor. При перемещении указателя мыши, выделены элементы HTML. **Стили** вкладке отображаются правила CSS для элемента.

<a id="_5_using_page"></a>

## <a name="use-page-inspector-to-make-changes-to-markup"></a>Использование инспектора страниц вносить изменения в разметку

Инспектор страниц позволяет найти разметки, местоположение которого могут быть неявными. Затем можно изменить разметку и см. в разделе результирующих изменений.

Чтобы увидеть это, нажмите кнопку **инспектировать** и прокрутите до нижней части страницы в окне инспектора страниц.

При перемещении указателя мыши в области нижнего колонтитула, инспектор страниц открывает \_файла Layout.cshtml и выделяет часть макета страницы, выбранный вами. Как вы видите, нижний колонтитул, определенный в файле макета, а не само представление.

![Нижние колонтитулы](using-page-inspector-in-aspnet-mvc/_static/image16.png)

Теперь наведите указатель мыши на строку с авторских прав <a id="a"> </a>Обратите внимание, что. В \_страницу Layout.cshtml, соответствующая строка будет выделена.

![Строка нижнего колонтитула об авторских правах, выделяемая подсветкой](using-page-inspector-in-aspnet-mvc/_static/image18.png)

Добавьте какой-нибудь текст в конец строки в \_файла Layout.cshtml.

&lt;p&gt;&amp;copy; @DateTime.Now.Year -Мое приложение ASP.NET MVC крут хотите знать?! &lt;/p&gt;

Теперь нажмите клавиши Ctrl + Alt + Ввод или щелкните панель обновления, чтобы просмотреть результаты в окне браузера инспектора страниц.

![My ASP.NET Application Rocks!](using-page-inspector-in-aspnet-mvc/_static/image20.png)

Вы наверняка считали, что нижнего колонтитула, определенные в файле Index.cshtml. Однако это оказалось в \_Layout.cshtml и инспектор страниц нашли его для вас.

## <a name="inspection-mode-and-the-html-window"></a>Режим инспектирования и окна HTML

Затем вы получите краткий обзор HTML-окну, и способом сопоставления элементов для вас.

Нажмите кнопку **инспектировать** чтобы перевести инспектор страниц в режиме инспектирования.

Щелкните в верхней части страницы «Ваша эмблема здесь». Изучении конкретный элемент более подробно, поэтому отображение в окне браузера больше не изменяется при перемещении указателя мыши.

Теперь наведите указатель мыши на **HTML** окна. При перемещении указателя мыши, инспектор страниц описаны сервисном **HTML** окне и выделяет соответствующий элемент в окне браузера.

![Окно HTML](using-page-inspector-in-aspnet-mvc/_static/image22.png)

Как и раньше, инспектор страниц открывает \_файла Layout.cshtml для вас в временную вкладку. Нажмите кнопку \_Layout.cshtml временную вкладку, а соответствующая разметка будет выделена &lt;заголовок&gt; раздел для вас:

![Выделенная разметка](using-page-inspector-in-aspnet-mvc/_static/image24.png)

<a id="_using_page_inspector"></a><a id="_7_previewing_css"></a>

## <a name="preview-css-changes-in-the-styles-window"></a>Просмотр изменений CSS в окне "Стили"

Затем будет использовать инспектор страниц **стили** окно, чтобы просмотреть изменения в CSS.

Нажмите кнопку **инспектировать** чтобы перевести инспектор страниц в режиме инспектирования.

В окне браузера инспектора страниц наведите указатель мыши на раздел «Домашняя страница» до **div.content оболочки** появится метка.

![При наведении div.content оболочки](using-page-inspector-in-aspnet-mvc/_static/image26.png)

Щелкните в разделе div.content оболочку, а затем наведите указатель мыши на **стили** окна. **Стили** окне показаны все правила CSS для данного элемента. Прокрутите вниз до селектор класса поиска .featured .content оболочки. Теперь снимите флажок для свойства цвет фона.

![Цвет фона очистить](using-page-inspector-in-aspnet-mvc/_static/image28.png)

Обратите внимание на то, как изменение выполняет предварительный просмотр сразу же в окне браузера инспектора страниц.

Снова установите флажок, затем дважды щелкните значение свойства и измените его на красный. Изменение немедленно показано:

![Красного цвета фона](using-page-inspector-in-aspnet-mvc/_static/image30.png)

**Стили** облегчает тестирование и предварительный просмотр CSS изменения, прежде чем зафиксировать изменения стиля делает окно листа сам.

<a id="css_auto_sync"></a>
## <a name="css-auto-sync"></a>Автосинхронизация CSS

> [!NOTE]
> Эта функция требует версии 1.3 инспектора страниц.


Автосинхронизация CSS позволяет напрямую изменить CSS-файл и увидеть изменения прямо в браузере инспектора страниц.

Нажмите кнопку **инспектировать** чтобы перевести инспектор страниц в режиме инспектирования.

В браузере инспектора страниц наведите указатель мыши на раздел «Домашняя страница» до **div.content оболочки** появится метка. Щелкните один раз, чтобы выбрать этот элемент.

**Стили** окне показаны все правила CSS для данного элемента. Прокрутите вниз до селектор класса поиска .featured .content оболочки. Щелкните «.featured .content оболочки». Инспектор страниц открывает CSS-файл, который определяет этот стиль (Site.css) и выделяет соответствующий стиль CSS.

![](using-page-inspector-in-aspnet-mvc/_static/image32.png)

Теперь измените значение `background-color` «красный». Изменение немедленно отображается в браузере инспектора страниц.

![](using-page-inspector-in-aspnet-mvc/_static/image34.png)

<a id="css_color_picker"></a>
## <a name="using-the-css-color-picker"></a>В палитре цветов CSS

Редактор CSS в Visual Studio 2012 содержит палитру цветов, позволяет легко выбрать и вставить цвета. Палитра содержит стандартную палитру цветов, поддерживает стандартные имена цветов, хэш-кодов, цвета RGB, RGBA, HSL и HSLA и ведет список цветов, которые наиболее недавно использовавшегося в документе.

В предыдущем разделе, вы изменили значение `background-color` свойства. Чтобы вызвать палитра цветов, поместите курсор после имени свойства и типа **#** или **rgb (**.

![Палитра цветов CSS](using-page-inspector-in-aspnet-mvc/_static/image36.png)

Щелкните цвет, чтобы выбрать его, или нажмите клавишу со стрелкой вниз и затем с помощью клавиш со стрелками влево и вправо для обхода цвета. При посещении цвет, предварительном просмотре соответствующее шестнадцатеричное значение:

![значение свойства цвет фона предварительного просмотра](using-page-inspector-in-aspnet-mvc/_static/image38.png)

Если цветовой полосы не имеет точного нужного цвета, можно использовать средство выбора цвета pop вниз. Чтобы открыть его, щелкните значок шеврона в правом конце цветовой полосы, или один или два раза нажмите клавишу со стрелкой вниз на клавиатуре.

![Средство выбора цвета CSS, Pop вниз](using-page-inspector-in-aspnet-mvc/_static/image40.png)

Выберите цвет из вертикальная черта справа. Это показывает градиента цвета в главном окне. Выбрать цвет непосредственно из вертикальной чертой, нажав клавишу ВВОД или щелкните любую точку в главном окне для выбора с большей точностью.

Есть ли цвет на экране компьютера, который вы хотите использовать (он не должен быть в пользовательском интерфейсе Visual Studio), его значение можно захватить, используя инструмент "Пипетка" в правом нижнем углу.

Также можно изменить непрозрачность цвета, переместив ползунок в нижней части палитры цветов. Способ изменения цвета значения в значения RGBA, так как формат RGBA может представлять непрозрачности.

После выбора цвета, нажмите клавишу ВВОД, а затем введите точку с запятой, чтобы завершить ввод цвет фона в *Site.css* файл.

<a id="map_dynamic_elements"></a>
## <a name="mapping-dynamic-page-elements-to-javascript"></a>Сопоставление динамические элементы страниц JavaScript

В современных веб-приложений элементы страницы часто создается динамически с помощью JavaScript. Это означает, что статические разметка отсутствует (HTML или Razor), соответствующий эти элементы страницы.

С версии 1.3 инспектор страниц теперь можно изменять элементы, которые были динамически добавлены на страницу обратно на соответствующий код JavaScript. Чтобы продемонстрировать эту функцию, мы будем использовать [одностраничное приложение (SPA) шаблона](../../../single-page-application/overview/introduction/knockoutjs-template.md).

> [!NOTE]
> Шаблон одностраничного приложения требуется [ASP.NET и веб-инструментами 2012.2](https://go.microsoft.com/fwlink/?LinkId=282650) обновления.


В Visual Studio, выберите **файл** &gt; **новый проект**. Слева разверните **Visual C#** выберите **Web**, а затем выберите **веб-приложение ASP.NET MVC 4**. Нажмите кнопку **ОК**.

В **создания проекта ASP.NET MVC 4** диалоговом окне выберите **одностраничного приложения**.

В обозревателе решений разверните **представления** папку и затем **Главная** папки. Файл Index.cshtml щелкните правой кнопкой мыши и выберите **просмотреть в инспекторе страниц**.

Первое, что это отображается в браузере инспектора страниц — это страница входа. Щелкните «Зарегистрироваться» и создайте имя пользователя и пароль. После регистрации, приложение вход будет выполнен и создает список дел с некоторых образцов элементов.

Нажмите кнопку **инспектировать** чтобы перевести инспектор страниц в режиме инспектирования. В браузере инспектора страниц щелкните один из элементов списка дел. Обратите внимание на то, что вместо выделены синим цветом, элемент выделены оранжевым цветом, с помощью «JS» рядом с именем элемента. Это означает, что элемент был создан динамически при помощи сценария.

![](using-page-inspector-in-aspnet-mvc/_static/image44.png)

Кроме того, оранжевый подчеркивание отображается на **стек вызовов** вкладки. Это означает, что **стек вызовов** панели находятся дополнительные сведения об элементе.

Щелкните **стек вызовов** вкладки. **Стек вызовов** области показан стек вызовов для вызова JavaScript, создавшего элемент. Вызывается на внешние библиотеки такие, как jQuery свернуты, можно легко увидеть вызовы к сценарию приложения.

![](using-page-inspector-in-aspnet-mvc/_static/image46.png)

Чтобы просмотреть полный стек, включая вызовы на внешние библиотеки можно развернуть узлы с меткой «Внешние библиотеки»:

![](using-page-inspector-in-aspnet-mvc/_static/image48.png)

Если щелкнуть элемент в стеке вызовов, Visual Studio открывает файл кода и выделяет соответствующий сценарий.

![](using-page-inspector-in-aspnet-mvc/_static/image50.png)
