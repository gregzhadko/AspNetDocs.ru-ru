# <a name="self-host-aspnet-web-api-1-c"></a>Самостоятельный веб-API ASP.NET 1 (C#)

по [Майк Уоссон](https://github.com/MikeWasson)

> В этом руководстве показано, как разместить веб-API внутри консольного приложения. Для веб-API ASP.NET не требуются службы IIS. Веб-API можно самостоятельно разместить в собственном процессе узла. 
> 
> **Новые приложения должны использовать OWIN для веб-API с самостоятельным размещением.** См. раздел [Использование OWIN для самостоятельного размещения веб-API ASP.NET 2](../hosting-aspnet-web-api/use-owin-to-self-host-web-api.md).

## <a name="create-the-console-application-project"></a>Создание проекта консольного приложения

Запустите Visual Studio и выберите **создать проект** на **начальной** странице. Либо в меню **файл** выберите **создать** , а затем — **проект**.

В области **шаблоны** выберите **Установленные шаблоны** и разверните узел  **C# визуального** элемента. В **разделе C#визуальный** элемент выберите **Windows**. В списке шаблонов проектов выберите **консольное приложение**. Присвойте проекту имя &quot;резидент&quot; и нажмите кнопку **ОК**.

![](self-host-a-web-api/_static/image1.png)

## <a name="set-the-target-framework-visual-studio-2010"></a>Настройка целевой платформы (Visual Studio 2010)

Если вы используете Visual Studio 2010, измените целевую платформу на .NET Framework 4,0. (По умолчанию шаблон проекта предназначен для [клиентского профиля .NET Framework](https://msdn.microsoft.com/library/cc656912.aspx#features_not_included_in_the_net_framework_client_profile).)

В обозреватель решений щелкните правой кнопкой мыши проект и выберите пункт **Свойства**. В раскрывающемся списке **Целевая платформа** измените целевую платформу на .NET Framework 4,0. При появлении запроса на применение изменений нажмите кнопку **Да**.

## <a name="install-nuget-package-manager"></a>Установка диспетчера пакетов NuGet

Диспетчер пакетов NuGet — это самый простой способ добавления сборок веб-API в проект non-ASP.NET.

Чтобы проверить, установлен ли диспетчер пакетов NuGet, откройте меню **Сервис** в Visual Studio. Если отображается пункт меню **Диспетчер пакетов NuGet**, то у вас есть диспетчер пакетов NuGet.

Чтобы установить диспетчер пакетов NuGet, выполните следующие действия.

1. Запустите среду Visual Studio.
2. Откройте меню **Средства** и выберите пункт **Расширения и обновления**.
3. В диалоговом окне **расширения и обновления** выберите в **сети**.
4. Если вы не видите "Диспетчер пакетов NuGet", в поле поиска введите "Диспетчер пакетов NuGet".
5. Выберите Диспетчер пакетов NuGet и нажмите кнопку **скачать**.
6. После завершения скачивания вам будет предложено установить.
7. После завершения установки может быть предложено перезапустить Visual Studio.

![](self-host-a-web-api/_static/image3.png)

## <a name="add-the-web-api-nuget-package"></a>Добавление пакета NuGet для веб-API

После установки диспетчера пакетов NuGet добавьте в проект пакет самообслуживания веб-API.

1. В меню **Сервис** выберите **Диспетчер пакетов NuGet**. *Примечание*. Если вы не видите этот пункт меню, убедитесь, что диспетчер пакетов NuGet установлен правильно.
2. Выберите **Управление пакетами NuGet для решения**
3. В диалоговом окне **Управление пакетами NugGet** выберите в **сети**.
4. В поле поиска введите &quot;Microsoft. AspNet. WebApi. резидент&quot;.
5. Выберите веб-API ASP.NET пакет самообслуживания и нажмите кнопку **установить**.
6. После установки пакета нажмите кнопку **Закрыть** , чтобы закрыть диалоговое окно.

> [!NOTE]
> Убедитесь, что установлен пакет с именем Microsoft. AspNet. WebApi. резидент, а не Аспнетвебапи. резидент.

![](self-host-a-web-api/_static/image4.png)
