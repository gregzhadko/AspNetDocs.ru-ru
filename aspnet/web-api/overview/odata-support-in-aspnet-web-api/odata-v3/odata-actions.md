# <a name="supporting-odata-actions-in-aspnet-web-api-2"></a>Корректирующие действия OData в веб-API 2 ASP.NET

по [Майк Уоссон](https://github.com/MikeWasson)

[Скачать завершенный проект](http://code.msdn.microsoft.com/ASPNET-Web-API-OData-cecdb524)

> В OData *действия* позволяют добавлять поведения на стороне сервера, которые легко не определены как операций CRUD в объектах. Некоторые способы для действий:
> 
> - Реализация сложных транзакциях.
> - Управление несколько сущностей за один раз.
> - Разрешение обновлений только на определенные свойства сущности.
> - Отправка данных на сервер, который не определен в сущности.
> 
> ## <a name="software-versions-used-in-the-tutorial"></a>Версии программного обеспечения, используемые в этом руководстве
> 
> 
> - Веб-API 2
> - OData версии 3
> - Entity Framework 6

**FunctionImport** элемент объявляет действие. Большинство полей говорят сами за себя, но два стоит обратить внимание:

- **Имеет значение IsBindable** означает, что действие может вызываться в целевой сущности, по крайней мере некоторое время.
- **IsAlwaysBindable** означает, что действие, всегда будут вызываться в целевой сущности.

Разница в том случае, что некоторые действия всегда будут доступны клиентам, но другие действия могут опираться на состояние сущности. Например предположим, что вы определяете действие «Приобрести». ## <a name="transient-actions"></a>Временные действия

Например в службе аренды видео, действие «Оформление заказа» не всегда доступен. (Оно зависит от доступен ли копия этого видео.) Этот тип действия вызывается *временных* действие.
