# <a name="adding-security-and-membership-to-an-aspnet-web-pages-razor-site"></a>Добавление безопасности и членства на сайте веб-страницы ASP.NET (Razor)

от [Tom фитзмаккен](https://github.com/tfitzmac)

> В этой статье объясняется, как защитить веб-сайт веб-страницы ASP.NET (Razor), чтобы некоторые страницы были доступны только тем пользователям, которые входят в систему. (Вы также узнаете, как создавать страницы, к которым может обращаться любой пользователь.)
> 
> **Что вы узнаете:** 
> 
> - Как создать веб-сайт со страницей регистрации и страницей входа, чтобы на некоторых страницах можно было ограничить доступ только к элементам.
> - Создание открытых и только членов страниц.
> - Определение ролей, которые представляют собой группы, имеющие различные разрешения безопасности на сайте, и назначение пользователей роли.
> - Использование CAPTCHA для предотвращения создания учетных записей членов автоматизированными программами (программы-роботы).
>   
> 
> Это функции ASP.NET, представленные в статье:
> 
> - Шаблон **сайта "начальный сайт** " WebMatrix.
> - Вспомогательный класс `WebSecurity` и `Roles`.
> - Вспомогательный метод `ReCaptcha`.

Вы можете настроить веб-сайт таким образом, чтобы пользователи могли войти &#8212; в него, чтобы сайт поддерживал *членство*. Это может быть полезно по многим причинам. Например, на сайте могут быть страницы, которые должны быть доступны только участникам. В некоторых случаях пользователям может потребоваться выполнить вход, чтобы отправить отзыв или оставить комментарий.

Даже если ваш веб-сайт поддерживает членство, пользователи не обязательно должны входить в систему, прежде чем использовать некоторые страницы на сайте. Пользователи, которые не вошли в систему, называются *анонимными пользователями*.

Пользователь может зарегистрироваться на веб-сайте и затем войти на сайт. Веб-сайту требуется имя пользователя (адрес электронной почты) и пароль для подтверждения того, что пользователи должны быть. Этот процесс входа и подтверждения удостоверения пользователя называется *проверкой подлинности*.

Вы можете настроить безопасность и членство различными способами.

- Если вы используете WebMatrix, простой способ — создать новый сайт на основе шаблона **начального сайта** . Этот шаблон уже настроен для безопасности и членства и уже содержит страницу регистрации, страницу входа и т. д.

    Сайт, созданный с помощью шаблона, также имеет возможность входа пользователей с помощью внешнего сайта, например Facebook, Google или Twitter.
- Если требуется добавить безопасность на существующий сайт или если вы не хотите использовать шаблон **начального сайта** , можно создать собственную страницу регистрации, страницу входа и т. д.

Эта статья посвящена первому варианту &mdash;, как добавить безопасность с помощью шаблона **начального сайта** . Он также предоставляет некоторые основные сведения о том, как реализовать собственную безопасность, а затем предоставляет ссылки на дополнительные сведения о том, как это сделать. Также есть сведения о том, как включить внешние имена входа, которые подробно описаны в отдельной статье.

## <a name="creating-website-security-using-the-starter-site-template"></a>Создание безопасности веб-сайта с помощью шаблона начального сайта

В WebMatrix можно использовать шаблон **начального сайта** для создания веб-сайта, который содержит следующее:

- База данных, которая используется для хранения имен пользователей и паролей членов.
- Страница регистрации, на которой могут регистрироваться анонимные (новые) пользователи.
- Страница входа и выхода.
- Страница восстановления и сброса пароля.

## <a name="additional-membership-functionality-in-the-site"></a>Дополнительные функции членства на сайте

Ваш сайт содержит другие функции, позволяющие пользователям управлять своими учетными записями. Пользователи могут выполнять следующие действия:

- Измените пароли. После входа в систему можно щелкнуть имя пользователя (ссылка). После этого они перемещаются на страницу, где можно создать новый пароль (*учетная запись/ChangePassword. cshtml*).
- Восстановление забытого пароля. На странице входа есть ссылка (**вы забыли пароль?** ), которая принимает на страницу (*Account/ForgotPassword. cshtml*), где можно ввести адрес электронной почты. Сайт отправляет им сообщение электронной почты со ссылкой, которую можно щелкнуть, чтобы задать новый пароль (*Account/PasswordReset. cshtml*).

Кроме того, можно разрешить пользователям входить в систему с помощью внешнего сайта, как описано далее.

## <a name="creating-a-members-only-page"></a>Создание страницы только для членов

В течение этого времени любой пользователь может перейти на любую страницу веб-сайта. Но может потребоваться, чтобы страницы были доступны только тем пользователям, которые вошли в систему (то есть к членам). ASP.NET позволяет создавать страницы, доступ к которым можно получить только членам, вошедшим в систему. Как правило, если анонимные пользователи пытаются получить доступ к странице только для членов, они перенаправляются на страницу входа.

Для защиты доступа к нескольким страницам можно сделать следующее:

- Добавьте проверку безопасности на каждую страницу.
- Создайте страницу *\_пажестарт. cshtml* в папке, в которой хранятся защищенные страницы, и добавьте в нее проверку безопасности. Страница *\_пажестарт. cshtml* действует как разновидность глобальной страницы для всех страниц в папке. Этот метод более подробно описан в статье [Настройка поведения в масштабах всего сайта для веб-страницы ASP.NET](https://go.microsoft.com/fwlink/?LinkId=202906#Using__PageStart.cshtml_to_Restrict_Folder_Access).

## <a name="creating-security-for-groups-of-users-roles"></a>Создание безопасности для групп пользователей (ролей)

Если на вашем веб-узле много членов, не рекомендуется проверять разрешения для каждого пользователя по отдельности, прежде чем разрешить им просмотр страницы. Вместо этого можно создать группы или *роли*, к которым принадлежат отдельные члены. Затем можно проверить разрешения на основе роли. В этом разделе вы создадите роль администратора &quot;&quot;, а затем создадите страницу, доступную для пользователей, входящих в эту роль.

Система членства ASP.NET настроена для поддержки ролей. Однако, в отличие от регистрации членства и входа в систему, шаблон **начального сайта** не содержит страниц, помогающих управлять ролями. (Управление ролями является административной задачей, а не пользовательской задачей.) Однако группы можно добавлять непосредственно в базу данных членства в WebMatrix.

## <a name="letting-users-log-in-using-an-external-site"></a>Предоставление пользователям возможности входа с помощью внешнего сайта

Шаблон **начального сайта** содержит код и разметку, позволяющие пользователям входить в систему с помощью Facebook, Windows Live, Twitter, Google или Yahoo. По умолчанию эта функция отключена. Общая процедура использования для входа пользователей, использующих эти внешние поставщики, заключается в следующем:

- Решите, какие внешние сайты требуется поддерживать.
- При необходимости перейдите на этот сайт и настройте приложение для входа. (Например, это необходимо сделать, чтобы разрешить имена для входа Facebook.)
- На сайте Настройте поставщик. В большинстве случаев нужно просто раскомментировать часть кода в файле *\_AppStart. cshtml* .
- Добавьте разметку на страницу регистрации, которая позволяет пользователям ссылаться на внешний сайт для входа в систему. Обычно можно скопировать требуемую разметку и немного изменить текст.

Пошаговые инструкции можно найти в разделе [Включение входа с внешних сайтов на веб-страницы ASP.net сайте](https://go.microsoft.com/fwlink/?LinkId=251969).

После входа пользователя из другого сайта пользователь возвращается на сайт и *связывается* с вашим сайтом. По сути, при этом создается запись членства на сайте для внешнего имени входа пользователя. Это позволяет использовать обычные средства членства (например, роли) с внешним именем входа.

## <a name="adding-security-to-an-existing-website"></a>Добавление безопасности на существующий веб-сайт

Процедура, описанная выше в этой статье, полагается на использование шаблона **начального сайта** в качестве основания для обеспечения безопасности веб-сайта. Если нецелесообразно начинать с шаблона **начального сайта** или копировать соответствующие страницы с сайта, основанного на этом шаблоне, можно реализовать тот же тип безопасности на своем сайте, написав его самостоятельно. Вы создадите такие же типы страниц, как регистрация, вход и т. д., а затем используете вспомогательные методы и классы для настройки членства.

Базовый процесс описан в записи блога [основной способ реализации безопасности ASP.NET Razor](http://mikepope.com/blog/DisplayBlog.aspx?permalink=2240). Большая часть работы выполняется с помощью следующих методов и свойств вспомогательного метода `WebSecurity`:

- [Вебсекурти. усерексистс](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.userexists(v=vs.99).aspx), [креатеусерандаккаунт](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.createuserandaccount(v=vs.99).aspx). Эти методы позволяют определить, зарегистрирован ли кто-то или нет.
- [Вебсекурти. прошли проверку подлинности](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.isauthenticated(v=vs.99).aspx). Это свойство позволяет определить, вошел ли текущий пользователь в систему. Это полезно для перенаправления пользователей на страницу входа, если они еще не вошли в систему.
- [Безопасность. Вход](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.login(v=vs.99).aspx), [безопасность. выход](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.logout(v=vs.99).aspx). Эти методы выполняют вход пользователя в систему.
- [Куррентусернаме](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.currentusername(v=vs.99).aspx). Это свойство полезно для отображения имени текущего пользователя, выполнившего вход в систему (если пользователь выполнил вход).
- [Конфирмаккаунт](https://msdn.microsoft.com/library/gg569286(v=vs.99).aspx). Этот метод полезен при настройке подтверждения электронной почты для регистрации. (Подробности описаны в записи блога [с помощью функции подтверждения веб-страницы ASP.NET безопасности](http://mikepope.com/blog/DisplayBlog.aspx?permalink=2267).)

Для управления ролями можно использовать классы [ролей](https://msdn.microsoft.com/library/gg538398(v=vs.99).aspx) и [членства](https://msdn.microsoft.com/library/gg569035(v=vs.99).aspx) , как описано в записи блога.
