Веб-сайта можно настроить таким образом, пользователи могут входить в него &#8212; то есть, чтобы сайт поддерживал *членства*. Это может быть полезно по многим причинам. Например у узла может быть страниц, которые должны быть доступны только для членов. В некоторых случаях может потребоваться пользователям входить в систему, чтобы отправить отзыв или комментарий.

Даже если веб-сайт поддерживает членство, пользователи не обязательно требуется, чтобы войти, прежде чем использовать некоторые из страниц на сайте. Пользователей, которые не вошли в известны как *анонимным пользователям*.

Пользователь может зарегистрировать на веб-сайта и затем можно войти на сайт. Веб-сайт требует имя пользователя (адрес электронной почты) и пароль, чтобы убедиться, что пользователи, выдают. Этот процесс входа и подтверждения удостоверения пользователя называется *проверки подлинности*.

Можно настроить безопасность и членства по-разному:

- Если вы используете WebMatrix, простой способ — создать в качестве нового сайта, на основе **начального сайта** шаблона. Этот шаблон уже настроен для обеспечения безопасности и членство и уже страницу регистрации, страница входа и т. д.

    Сайт, созданный с помощью шаблона есть параметры, чтобы разрешить пользователям вход с использованием на внешний сайт, например, Google, Facebook или Twitter.
- Если вы хотите добавить к существующему сайту безопасности, или в том случае, если вы не хотите использовать **начального сайта** шаблона, можно создать собственные страницы регистрации, страница входа и т. д.

Эта статья посвящена первый вариант &mdash; как для обеспечения дополнительной безопасности с помощью **начального сайта** шаблона. Он также предоставляет некоторые основные сведения о том, как реализовать собственную безопасность, а затем предоставляет ссылки на дополнительные сведения о том, как это сделать. Имеется также сведения о включении внешних имен входа, который описан более подробно в отдельной статье.

## <a name="creating-website-security-using-the-starter-site-template"></a>Создание веб-сайт безопасности, используя шаблон начального сайта

В WebMatrix, вы можете использовать **начального сайта** шаблона для создания веб-сайт со следующим содержимым:

- База данных, который используется для хранения имен пользователей и паролей для членов.
- Страницу регистрации, где можно зарегистрировать анонимным пользователям (новая).
- Страница входа и выхода.
- Страница восстановления и сброса пароля.

## <a name="additional-membership-functionality-in-the-site"></a>Дополнительные членством на сайте

Веб-узел содержит другие функциональные возможности, которая позволяет пользователям управлять учетной записью. Пользователей можно сделать следующее:

- Измените свои пароли. После входа в систему, они щелкните имя пользователя (она является ссылкой). Это ведет на страницу которой они могут создать новый пароль (*Account/ChangePassword.cshtml*).
- Восстановите забытый пароль. На странице входа, есть ссылка (**забыли пароль?**), принимающий пользователей на страницу (*Account/ForgotPassword.cshtml*) где они могут ввести адрес электронной почты. Сайт отправляет их сообщения электронной почты, содержащее ссылку, они можно щелкнуть, чтобы задать новый пароль (*Account/PasswordReset.cshtml*).

Можно также разрешить пользователям вход с использованием внешнего сайта, а также можно как описано ниже.

## <a name="creating-a-members-only-page"></a>Создание страницы только для членов

На данный момент, любой пользователь можно перейти к любой странице веб-сайта. Но вам может потребоваться страниц, которые доступны только пользователям, вошедшим в (то есть к членам). ASP.NET позволяет создавать страницы, которые можно осуществлять только члены вошедшего в систему. Как правило если анонимные пользователи пытаются получить доступ к странице, доступной только для членов, можно перенаправить их на страницу входа.

2. Назовите новую папку *члены*.
3. Внутри *члены* папки, создайте новую страницу и назвали его *MembersInformation.cshtml*.

## <a name="creating-security-for-groups-of-users-roles"></a>Создание системы безопасности для групп пользователей (роли)

Если сайт имеет много членов, не эффективен для проверки разрешений для каждого пользователя по отдельности, перед ее см. в разделе страницы. Что можно делать вместо этого — создать группы, или *ролей*, что отдельные члены принадлежат. Затем можно проверить разрешения на основе роли. В этом разделе вы создадите &quot;администратора&quot; роли и затем создать страницу, которая доступна для пользователей, находящихся в (принадлежащих) этой роли.

Система членства ASP.NET предназначена для поддержки ролей. Однако в отличие от членства регистрации и входа **начального сайта** шаблон не содержит страницы, которые помогут вам управлять ролями. 13. В адресной строке браузера добавьте *AdminOnly* в URL-адрес. (Другими словами, запрос *AdminOnly.cshtml* файл.) Вы будете перенаправлены *AdminError.cshtml* странице, так как вы не вошли в настоящее время в качестве пользователя в &quot;администратора&quot; роли.
14. Вернитесь к *Default.cshtml* и войдите в систему как пользователь, добавленный к &quot;администратора&quot; роли.
15. Перейдите к *AdminOnly.cshtml* страницы. Это время вы видите страницу.

## <a name="letting-users-log-in-using-an-external-site"></a>Позволяя пользователям вход с использованием внешнего сайта

**Начального сайта** шаблон включает в себя код и разметку, которая позволяет пользователям выполнить вход с помощью Facebook, Windows Live, Twitter, Google или Yahoo. По умолчанию эта функция не включена. Общая процедура использования, позволяя пользователям вход с использованием этих внешних поставщиков это:

- Определите, какие из внешних сайтов требуется поддерживать.
- При необходимости перехода на узел и настройка приложения для входа. (Например, необходимо сделать это, чтобы разрешить вход в Facebook.)
- На узле Настройка поставщика. В большинстве случаев достаточно раскомментируйте код в  *\_AppStart.cshtml* файла.
- Добавьте разметку на страницу регистрации, который позволяет пользователям ссылку на внешний сайт, для входа в систему. Обычно можно скопировать разметки требуется и немного измените текст.

Пошаговые инструкции можно найти в разделе [Включение имени входа с внешних сайтов на сайте веб-страниц ASP.NET](https://go.microsoft.com/fwlink/?LinkId=251969).

После входа пользователя с другого сайта, пользователь возвращается на сайт и *связывает* этому имени входа с помощью веб-узла. По сути это создает запись членства на узле для внешней учетной записи пользователя. Это позволяет использовать обычные средства членства (такие как роли) с внешней учетной записи.

## <a name="adding-security-to-an-existing-website"></a>Добавление безопасности существующего веб-сайта

Процедуру, описанную ранее в этой статье полагается на использование **начального сайта** шаблон в качестве основы для безопасности веб-сайта. Если это не так удобно для запуска из **начального сайта** шаблон или скопировать соответствующие страницы с сайта на основе этого шаблона, можно реализовать тот же тип безопасности на сайте создав его самостоятельно. Создать те же типы страниц, регистрации, входа и т. д. и затем использовать вспомогательные функции и классы для настройки членства.

Базовый процесс описан в записи блога [самый простой способ реализации безопасности ASP.NET Razor](http://mikepope.com/blog/DisplayBlog.aspx?permalink=2240). Большая часть работы выполняется с помощью следующих методов и свойств `WebSecurity` вспомогательный:

- [WebSecurty.UserExists](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.userexists(v=vs.99).aspx), [WebSecurity.CreateUserAndAccount](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.createuserandaccount(v=vs.99).aspx). Эти методы позволяют определить, кто-то уже зарегистрирован ли и их регистрации.
- [WebSecurty.IsAuthenticated](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.isauthenticated(v=vs.99).aspx). Это свойство позволяет определить, вошел ли текущий пользователь. Это полезно для перенаправления пользователей на страницу входа, если они еще не зарегистрировались.
- [WebSecurity.Login](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.login(v=vs.99).aspx), [WebSecurity.Logout](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.logout(v=vs.99).aspx). Эти методы входа пользователя, in или out.
- [WebSecurity.CurrentUserName](https://msdn.microsoft.com/library/webmatrix.webdata.websecurity.currentusername(v=vs.99).aspx). Это свойство полезно для отображения имени текущего пользователя вошедшего в систему (когда пользователь входит в систему).
- [WebSecurity.ConfirmAccount](https://msdn.microsoft.com/library/gg569286(v=vs.99).aspx). Этот метод полезен в том случае, если вы настроили подтверждение по электронной почте для регистрации. (Дополнительные сведения см. в записи блога [используете функции подтверждения для обеспечения безопасности веб-страниц ASP.NET](http://mikepope.com/blog/DisplayBlog.aspx?permalink=2267).)

Для управления ролями, можно использовать [ролей](https://msdn.microsoft.com/library/gg538398(v=vs.99).aspx) и [членства](https://msdn.microsoft.com/library/gg569035(v=vs.99).aspx) классов, как описано в записи блога.
