> [!TIP] 
> 
> **Типы данных и методов метод As()**
> 
> Мы только вышесказанное, при создании переменной используется ключевое слово var. Весь материал, однако не. Чтобы управлять значениями — Добавление номеров, или объединения строк, сравнения дат или проверки на true или false — C# должно работать с соответствующие внутреннего представления значения. C# можно *обычно* выяснить это представление, которое должно быть (то есть что *тип* данные) зависимости от выполняемых задач со значениями. Однако он не предусмотрена. В противном случае вам придется оказать помощь, явно указав как C# должен представлять данные. Метод AsBool делает это, он сообщает C# что строковое значение «true» или «false» должно рассматриваться как значение типа Boolean. Существуют аналогичные методы для представления строк как другие типы, например AsInt (Power BI как целое число), AsDateTime (обрабатывать как даты и времени), AsFloat (обрабатывать как число с плавающей запятой) и т. д. При использовании их в качестве методов (), если C# не может представлять строковое значение, по запросу, вы увидите ошибку.

### <a name="summary-of-your-conditional-logic"></a>Сводка условной логики

- Сообщение переменная инициализируется в строку по умолчанию («This is в первый раз...»).

- Переменная showMessage устанавливается равным false.
- Если строка запроса содержит? Показать = true, showMessage переменной задается значение true.
- В разметке, если значение равно true, showMessage &lt;p&gt; элемент отображается, показывающий значение сообщения. (Если showMessage имеет значение false, ничего не отображается на этом этапе в разметке.)
- В разметке, если запрос post &lt;p&gt; элемент отображается, который отображает дату и время.

## <a name="installing-a-helper-displaying-a-gravatar-image"></a>Установка вспомогательного метода (отображение изображения Gravatar)

Некоторые задачи, которые пользователи часто стремятся сделать на веб-страницах, требуют большого объема кода или требует дополнительных знаний. Примеры: отображение диаграммы для данных; Размещение Facebook «Как» на странице; При отправке сообщения с веб-сайта; Обрезание или изменение размеров изображений; с помощью PayPal для вашего сайта. Чтобы упростить процесс для выполнения таких задач, веб-страниц ASP.NET позволяет использовать *вспомогательные функции*. Вспомогательные функции являются компонентами, устанавливаемый для сайта, и для свойства, которые позволяют выполнять типовые задачи с помощью всего нескольких строк кода Razor.

Веб-страницы ASP.NET имеет несколько вспомогательных функций, встроенных. Тем не менее многие вспомогательные функции доступны в пакетах (надстройки), которые предоставляются с помощью диспетчера пакетов NuGet. NuGet позволяет выбрать пакет установки, а затем она отвечает за все аспекты установки.

В этой части руководства вы установите вспомогательный объект, которое служит для отображения изображения Gravatar («глобально распознанных аватар»). Вы узнаете следующее. Один, как найти и установить вспомогательный объект. Вы также узнаете, как вспомогательный объект позволяет легко выполнить задачи, в противном случае необходимо сделать с помощью большого объема кода, вам пришлось бы писать самостоятельно.

Вы можете зарегистрировать свои собственные Gravatar на веб-сайте Gravatar в [ http://www.gravatar.com/ ](http://www.gravatar.com/), но не обязательно, чтобы создать учетную запись Gravatar для выполнения этой части руководства.

В WebMatrix щелкните **NuGet** кнопки.

![Коллекция NuGet диалоговое окно, в WebMatrix](intro-to-web-pages-programming/_static/image7.png)

Это запускает диспетчер пакетов NuGet и отображает доступные пакеты. (Не все пакеты, вспомогательные функции; некоторые добавления функций в WebMatrix, сам, некоторые дополнительные шаблоны и т. д.) Может появиться сообщение об ошибке о несовместимости версий. Можно игнорировать это сообщение об ошибке, щелкнув **ОК** и работой с этим учебником.

![Коллекция NuGet диалоговое окно, в WebMatrix](intro-to-web-pages-programming/_static/image8.png)

В поле поиска введите «вспомогательные методы asp.net». NuGet отображаются пакеты, соответствующие условиям поиска.

![Коллекция NuGet в WebMatrix, показывающий пакетов](intro-to-web-pages-programming/_static/image9.png)

Выберите **библиотеку ASP.NET** пакета, а затем нажмите кнопку **установить** запустить программу установки. Выберите **Да** при запросе, если вы хотите установить пакет и примите условия, чтобы завершить установку.

Вот и все. NuGet загружает и устанавливает всем, включая все дополнительные компоненты, которые могут потребоваться (*зависимости*).

Если для какой-либо причине необходимо удалить вспомогательный объект, процесс очень похож. Нажмите кнопку **NuGet** , выберите элемент **установленные** вкладку и выберите пакет, который требуется удалить.
