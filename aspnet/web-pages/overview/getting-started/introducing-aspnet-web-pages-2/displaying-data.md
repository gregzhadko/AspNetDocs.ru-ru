## <a name="a-very-brief-introduction-to-databases"></a>Очень краткое введение в базы данных

В этом учебнике приводятся только briefest Знакомство с базами данных. Если у вас есть опыт базы данных, этот короткий раздел можно пропустить.

База данных содержит одну или несколько таблиц, содержащих сведения &mdash; например, таблицы, для клиентов, заказов и поставщиков или для учащихся, преподавателей, классы и оценок. Структурно таблицу базы данных — электронной таблицы. Представьте себе типичный адресной книги. Для каждой записи в адресной книге (то есть для каждого пользователя) у вас есть несколько блоков информации, такие как имя, Фамилия, адрес, адрес электронной почты и номер телефона.

![Образец таблицы базы данных в виде простой таблицы](displaying-data/_static/image2.png)

(Строки, иногда называются *записей*, и столбцы, иногда называются *поля*.)

Для большинства таблиц базы данных она должна содержать столбец, содержащий уникальное значение, например номер заказчика, номер счета и т. д. Это значение называется таблицы *первичный ключ*, и использовать его для идентификации каждой строки в таблице. В примере идентификатор столбца является первичным ключом для адресной книги, показанный в предыдущем примере.

Большую часть работы, как на веб-приложений состоит из считывания информации из базы данных и его отображения на странице. Вы также часто получать информацию от пользователей и добавить его к базе данных, или вы измените записей, которые уже находятся в базе данных. (Мы обсудим все эти операции в этот набор.)

Работа базы данных может быть невероятно сложной и может потребовать специальных знаний. Для данного руководства Однако вам нужно изучить только основные понятия, которые все объясняются при переходе.

## <a name="creating-a-table"></a>Создание таблицы

На ленте щелкните **новую таблицу**. WebMatrix откроется в конструкторе таблиц в новой вкладке. (Новая таблица недоступен, если убедитесь, что новая база данных выбрана в представлении в виде дерева в левой части.)

![Кнопка «Новую таблицу» в ленте WebMatrix](displaying-data/_static/image5.png)

В текстовом поле вверху (где водяной знак говорит: «Введите имя таблицы») введите «Movies».

![Ввод имени таблицы в конструкторе базы данных WebMatrix](displaying-data/_static/image6.png)

Области под имя таблицы является определение отдельных столбцов. Для *фильмы* таблицы в этом руководстве вы создадите только несколько столбцов: *Идентификатор*, *Title*, *жанр*, и *год*.

В **имя** введите «ID». Введенное здесь активирует все элементы управления для нового столбца.

Вкладки **тип данных** а выберите **int**. Это значение указывает, что столбец Идентификаторов будет содержать целочисленные данные (числа).

> [!NOTE]
> Мы не задаю ее любой см. Здесь (гораздо), но можно использовать стандартные жесты Windows клавиатуры для перехода в этой сетке. Например может переместиться между полями, вы можно просто начните ввод для выбора элемент в списке и так далее.

Клавиши табуляции **значение по умолчанию** поле (то есть, оставьте пустым). Вкладки **— первичный ключ** флажок и выберите его. Этот параметр предписывает базе данных, *идентификатор* столбец будет содержать данные, которые определяют отдельные строки. (То есть каждая строка будет иметь уникальное значение в столбца идентификатора, который можно использовать для поиска этой строки.)

Выберите **Is Identity** параметр. Этот параметр предписывает базе данных, он должен автоматически сгенерировать следующий последовательный номер для каждой новой строки. ( **Is Identity** работает параметр только в том случае, если вы выбрали также **— первичный ключ** параметр.)

В следующей строке сетки щелкните или нажмите клавишу Tab дважды, чтобы завершить текущую строку. Либо жестов сохраняет текущую строку и начинается следующий. Обратите внимание, что **значение по умолчанию** столбец теперь говорит **Null**. (Null значение по умолчанию для значения по умолчанию, так сказать.)

Завершив определение нового **идентификатор** столбец, конструктор будет выглядеть на этом рисунке:

![Конструктор баз данных WebMatrix, определив идентификатор столбца для таблицы фильмы](displaying-data/_static/image7.png)

Чтобы создать в следующий столбец, щелкните поле в **имя** столбца. Введите «Title» для столбца, а затем выберите **nvarchar** для **тип данных** значение. «Var» часть **nvarchar** сообщает базы данных, что данные для этого столбца будут строку, размер которых может меняться от записи к записи. (Префикс «n» представляет «национальные», которое указывает, что поле может содержать символьные данные для любого алфавит или системы письма — то есть поле содержит данные в Юникоде.)

При выборе **nvarchar**, откроется другое окно для ввода максимальную длину для поля. Введите 50 на предположении, что не название фильма, которым вы будете работать в этом руководстве будут длиннее 50 символов.

Пропустить **значение по умолчанию** и очистить **Разрешить значения NULL** параметр. Вы не хотите базы данных, фильмы для ввода в базу данных, на которых не установлена.

Когда все готово и перемещение на следующую строку, конструкторе выглядит как на этом рисунке:

![Конструктор баз данных WebMatrix после определения заголовка столбца для таблицы фильмы](displaying-data/_static/image8.png)

Создание другого столбца с именем «Год». Тип данных, выберите **nchar** (не **nvarchar**) и задающий длину равной 4. Для года вы собираетесь использовать 4-значное число, например «1995» или «2010», поэтому не требуется столбец переменного размера.

## <a name="displaying-data-in-a-page-by-using-the-webgrid-helper"></a>Отображение данных на странице с помощью вспомогательного элемента WebGrid

Для отображения данных на странице, вы собираетесь использовать `WebGrid` вспомогательный. Этот вспомогательный выдаются в виде сетки или таблицы (строки и столбцы). Как вы увидите, вы будете может выделить сетки с помощью форматирования и других функций.

Для запуска в сетке, вам придется написать несколько строчек кода. Эти несколько строк будет использоваться в качестве своего рода шаблон для почти всех доступа к данным, как и в этом руководстве.

> [!NOTE]
> Фактически существует несколько вариантов для отображения данных на странице; `WebGrid` вспомогательный — один. Мы выбрали его в этом руководстве, поскольку это самый простой способ отображения данных и является достаточно гибким. В рамках следующего учебного курса вы увидите, как использовать дополнительные «ручной» способ работы с данными в странице, которая обеспечивает более прямой контроль над тем, как для отображения данных.

В левой области в WebMatrix, щелкните **файлы** рабочей области.

Является новой базы данных, созданную в *приложения\_данных* папки. Если папка не существует, WebMatrix он создан для новой базы данных. (Папка может существовать Если вспомогательные функции были установлены ранее.)

В представлении в виде дерева выберите корень веб-сайта. Необходимо выбрать корневой веб-сайта; в противном случае новый файл могут быть добавлены в приложение\_папку данных.

На ленте щелкните **New**. В **выберите тип файла** выберите **CSHTML**.

В **имя** окне имя новой страницы «Movies.cshtml»:

![«Выберите тип файла» диалоговое окно страницы «Фильмы»](displaying-data/_static/image17.png)

Нажмите кнопку **ОК** кнопки. WebMatrix открывает новый файл с одними элементами схемы в нем. Сначала вы будете писать код, нужно получить данные из базы данных. Затем вы добавите разметки страницы, чтобы собственно, отображения данных.

### <a name="writing-the-data-query-code"></a>Написание кода запроса данных

В верхней части страницы между `@{` и `}` символы, введите следующий код. (Убедитесь, что вы введете этот код между открывающей и закрывающей фигурными скобками.)

[!code-csharp[Main](displaying-data/samples/sample1.cs)]

Первая строка открывает базы данных, который был создан ранее, который всегда является первым шагом перед действия с базой данных. Вы указываете `Database.Open` метод имя базы данных, чтобы открыть. Обратите внимание, что вы не включаете *.sdf* в имени. `Open` Метод предполагает, что он ищет *.sdf* файла (то есть *WebPagesMovies.sdf*) и что *.sdf* файл находится в *приложения\_ Данные* папки. (Ранее было отмечено, *приложения\_данных* папка является зарезервированным; этот сценарий является одно из мест, где ASP.NET делает предположения о существовании.)

При открытии базы данных, ссылку на него помещается в переменную с именем `db`. (Который может быть присвоено любое имя.) `db` Переменная доступна как в результате получается взаимодействия с базой данных.

Вторая строка фактически выполняет выборку данных базы данных с помощью `Query` метод. Обратите внимание на то, как работает этот код: `db` переменная имеет ссылку на открытую базу данных, и вы вызываете `Query` метод с помощью `db` переменной (`db.Query`).

Сам запрос — это SQL `Select` инструкции. (Немного Общие сведения о SQL, ознакомьтесь с описанием в более поздней версии.) В операторе `Movies` определяет таблицу для запроса. `*` Знак означает, что запрос должен возвращать все столбцы из таблицы. (Можно также указать столбцы по отдельности, разделенных запятыми.)

Результаты запроса, если таковые имеются, возвращаются и становятся доступными в `selectedData` переменной. Опять же переменная может быть присвоено любое имя.

Наконец, третья строка сообщает ASP.NET, что вы хотите использовать экземпляр `WebGrid` вспомогательный. Создании (*создать экземпляр*) вспомогательный объект, с помощью `new` ключевое слово и передайте его в результатах запроса с помощью `selectedData` переменной. Новый `WebGrid` объекта вместе с результатами запроса к базе данных, становятся доступными в `grid` переменной. Полученный результат вам потребуется чуть фактически выводит данные на странице.

На этом этапе база данных открыта, вы азы данных требуется, и после подготовки `WebGrid` вспомогательное приложение с помощью этих данных. Следующий шаг — создание разметки на странице.

> [!TIP] 
> 
> **Язык структурированных запросов (SQL)**
> 
> SQL — это язык, который используется в большинство реляционных баз данных для управления данными в базе данных. Она включает команды, которые позволяют получать данные и обновить его, и, которые позволяют создавать, изменять и управлять данными в таблицах базы данных. SQL отличается от языка программирования, (например C#). С помощью SQL указать базу данных, то, что нужно, и это задание базы данных, чтобы выяснить, как получить данные или выполнять задачи. Ниже приведены примеры некоторых команд SQL и что они делают.
> 
> `Select * From Movies`
> 
> `SELECT ID, Name, Price FROM Product WHERE Price > 10.00 ORDER BY Name`
> 
> Первый `Select` инструкция возвращает все столбцы (определяется `*`) из *фильмы* таблицы.
> 
> Второй `Select` инструкция извлекает идентификатор, имя и цена столбцы из записей в *продукта* таблицы, значение столбцов которых цена — более чем 10. Эта команда возвращает результаты в алфавитном порядке на основе значений имени столбца. Если нет записей, соответствующих критериям цена, команда возвращает пустой набор.
> 
> `INSERT INTO Product (Name, Description, Price) VALUES ('Croissant', 'A flaky delight', 1.99)`
> 
> Эта команда вставит новую запись в *продукта* таблице, задание для 1,99 столбце имен, чтобы «Круассан», столбца описания «Ненадежных удовлетворят A», а также цену.
> 
> Обратите внимание на то, что при указании нечисловое значение, значение заключено в одинарные кавычки (не двойные кавычки, как в C#). Можно использовать эти кавычки вокруг значений текст или даты, но не вокруг чисел.
> 
> `DELETE FROM Product WHERE ExpirationDate < '01/01/2008'`
> 
> Эта команда удаляет записи в *продукта* таблицы, в которой столбец даты окончания срока действия более ранняя, чем 1 января 2008 г. (Команда предполагает, что *продукта* таблица имеет такой столбец, конечно.) Здесь вводится дата в формате мм/дд/гггг, но должно вводиться в формате, который используется для используемого языкового стандарта.
> 
> `Insert` И `Delete` команды не возвращают результирующие наборы. Вместо этого они возвращают число, которое указывает, сколько записей, затронутых командой.
> 
> Для некоторых из этих операций (например, вставка и удаление записей) процесс, который запрашивает операцию должен иметь соответствующие разрешения в базе данных. Именно для рабочих баз данных, обычно требуется ввести имя пользователя и пароль при подключении к базе данных.
> 
> Существует множество команд SQL, но все они соответствуют шаблону, как и команды, которые вы видите здесь. Команды SQL можно использовать для создания таблиц баз данных, подсчитать число записей в таблице, расчета цен и выполнять многие другие операции.
