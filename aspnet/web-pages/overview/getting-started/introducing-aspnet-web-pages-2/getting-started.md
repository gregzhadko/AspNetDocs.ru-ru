## <a name="what-is-aspnet-web-pages"></a>Что такое веб-страницах ASP.NET?

Веб-страницы ASP.NET — это платформа, можно использовать для создания динамических веб-страниц. Простой веб-страницы HTML является статическим. его содержимое определяется предопределенной разметку HTML на странице. Динамических страниц, таких как те, которые можно создать с веб-страниц ASP.NET позволяют создавать содержимое страницы на ходу, с помощью кода.

Динамические страницы позволяют выполнять различные действия. Можно запроса у пользователя входных данных с помощью формы и затем изменить на странице отображается или его внешний вид. Можно получить информацию от пользователя, сохраните его в базу данных и затем перечислить его позже. Вы можете отправлять сообщения электронной почты из веб-узла. Можно взаимодействовать с другими службами в Интернете (например, служба сопоставления) и создания страниц, интегрировать данные из этих источников.

## <a name="what-is-webmatrix"></a>Что такое WebMatrix

WebMatrix — это средство, которое интегрируется в редакторе веб-страницы, программа базы данных, веб-сервер для тестирования страниц и функций для публикации веб-сайта к Интернету. WebMatrix — бесплатная, и ее легко установить и простой в использовании. (Она также работает для простого HTML-страниц, а также для других технологий, как PHP).

Фактически не *имеют* использовать WebMatrix для работы с веб-страниц ASP.NET. Можно создавать страницы с помощью текстового редактора, например и тестировать страницы с помощью веб-сервера, у вас есть доступ к. Тем не менее WebMatrix упрощает все очень просто, поэтому эти руководства будут использовать WebMatrix на протяжении всего.

## <a name="testing-the-page"></a>Тестирование страницы

В **файлы** рабочей области, щелкните правой кнопкой мыши *HelloWorld.cshtml* странице и нажмите кнопку **запустить в браузере**.

![Запуск страницы с помощью на ленте WebMatrix](getting-started/_static/image18.png)

WebMatrix запускает встроенный веб-сервер (IIS Express), можно использовать для тестирования страниц на компьютере. (Без IIS Express в WebMatrix, пришлось бы где-нибудь опубликовать страницу веб-сервера, прежде чем можно протестировать его.) Откроется страница в браузере по умолчанию.

![&quot;Здравствуй, мир&quot; страница, выполняемая в браузере](getting-started/_static/image19.png)

Обратите внимание, что при тестировании страницы WebMatrix, URL-адрес в браузере примерно `http://localhost:33651/HelloWorld.cshtml.` имя *localhost* ссылается на локальном сервере, это означает, что страница выдается веб-сервером, на своем компьютере. Как уже отмечалось, в WebMatrix имеются программы веб-сервера, с именем IIS Express, которая запускается при запуске страницы.

Число после *localhost* (например, *localhost:33651*) ссылается на *номер порта* на компьютере. Это число «канал», IIS Express для этого конкретного веб-сайта. Номер порта выбирается в произвольном порядке в диапазоне от 1024 до 65 536 при создании веб-узла, и оно отличается для каждого узла, можно создать. (Если вы тестируете свой собственный узел, номер порта почти наверняка будет другой номер, чем 33561.) С помощью другой порт для каждого веб-сайта, IIS Express можно хранить прямых какие из ваших сайтов, он взаимодействует.

Позже при публикации узла на общедоступный веб-сервере, вы больше не увидите *localhost* в URL-адрес. В таком случае вы увидите более типичный URL-адресу вида `http://myhostingsite/mywebsite/HelloWorld.cshtml` или любой страницы. Вы узнаете больше о публикации узла далее в этой серии руководств.

## <a name="this-is-what-aspnet-web-pages-are-about"></a>Это веб-страницы ASP.NET являются о

Когда вы читаете, что веб-страниц ASP.NET создает динамического веб-содержимого, теперь вы увидели здесь является идея. Страница, которую вы только что создали содержит же разметку HTML, вы уже видели. Он также может содержать код, который может выполнять любые задачи. В этом примере как нетривиальная задача получения текущей даты и времени. Как видно, вы можете перемежать код и HTML-код для получения выходных данных на странице. Когда кто-то запрашивает *.cshtml* страницу в браузере, ASP.NET обрабатывает страницу, хотя она по-прежнему руки веб-сервера. ASP.NET вставляет результат выполнения кода (если таковые имеются) в страницу как HTML. По завершении обработки кода, ASP.NET отправляет целевую страницу в браузер. Все, что когда-либо получает браузер является HTML. Ниже приведена схема:

![Концептуальный алгоритм как ASP.NET динамически создает HTML-код](getting-started/_static/image21.png)

Идея проста, но есть множество интересных задач, которые могут выполнять код, и существует много интересных способов, в которых можно динамически добавлять HTML-содержимого на страницу. И ASP.NET *.cshtml* страницы, как и любой HTML-страницы могут также включать код, выполняемый в браузере, сам (код JavaScript и jQuery). Рассмотрим все это в рамках этого учебного курса и последующие условия.
