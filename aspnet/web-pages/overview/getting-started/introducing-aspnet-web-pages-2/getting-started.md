## <a name="what-is-aspnet-web-pages"></a>Что такое веб-страницы ASP.NET?

Веб-страницы ASP.NET — это платформа, которую можно использовать для создания динамических веб-страниц. Простая веб-страница HTML является статической. его содержимое определяется фиксированной разметкой HTML, которая находится на странице. Динамические страницы, такие как созданные с помощью веб-страницы ASP.NET позволяют создавать содержимое страницы на лету с помощью кода.

Динамические страницы позволяют выполнять все виды действий. Можно попросить пользователя ввести данные с помощью формы, а затем изменить отображаемую страницу или ее вид. Можно получить сведения от пользователя, сохранить его в базе данных, а затем перечислить в дальнейшем. Вы можете отправлять электронную почту с сайта. Вы можете взаимодействовать с другими службами в Интернете (например, со службой сопоставления) и создавать страницы, которые интегрируют информацию из этих источников.

## <a name="what-is-webmatrix"></a>Что такое WebMatrix?

WebMatrix — это средство, объединяющее редактор веб-страниц, служебную программу для работы с базами данных, веб-сервер для тестирования страниц и функции для публикации веб-сайта в Интернете. WebMatrix бесплатный, и его легко установить и легко использовать. (Он также работает только для обычных HTML-страниц, а также для других технологий, таких как PHP.)

В действительности для работы с веб-страницы ASP.NET *не нужно использовать* WebMatrix. Можно создавать страницы с помощью текстового редактора, например, и тестировать страницы с помощью веб-сервера, к которому у вас есть доступ. Однако WebMatrix делает ее очень простой, поэтому в этих учебниках будет использоваться WebMatrix по всему.

## <a name="testing-the-page"></a>Тестирование страницы

В рабочей области **файлы** щелкните правой кнопкой мыши страницу *HelloWorld. cshtml* и выберите пункт **запустить в браузере**.

![Запуск страницы с помощью кнопки "выполнить" на ленте WebMatrix](getting-started/_static/image18.png)

WebMatrix запускает встроенный веб-сервер (IIS Express), который можно использовать для тестирования страниц на компьютере. (Без IIS Express в WebMatrix вам пришлось бы опубликовать страницу на веб-сервере, прежде чем ее можно будет протестировать.) Страница отобразится в браузере по умолчанию.

![&quot;Hello World &quot; страница, выполняемая в браузере](getting-started/_static/image19.png)

Обратите внимание, что при тестировании страницы в WebMatrix URL-адрес в браузере выглядит примерно так: `http://localhost:33651/HelloWorld.cshtml.` имя *localhost* относится к локальному серверу. Это означает, что страница обслуживается веб-сервером на вашем компьютере. Как уже отмечалось, WebMatrix включает в себя программу веб-сервера с именем IIS Express, которая выполняется при запуске страницы.

Число после *localhost* (например, *localhost: 33651*) указывает на *номер порта* на компьютере. Это номер "канала", который IIS Express использует для этого конкретного веб-сайта. Номер порта выбирается случайным образом из диапазона от 1024 до 65536 при создании сайта и отличается для каждого создаваемого сайта. (При тестировании собственного сайта номер порта почти наверняка будет отличаться от числа 33561.) Используя другой порт для каждого веб-сайта, IIS Express может быть в курсе, к какому из сайтов он обращается.

Позже при публикации сайта на общедоступном веб-сервере в URL-адресе не будет отображаться *localhost* . На этом этапе вы увидите более типичный URL-адрес, такой как `http://myhostingsite/mywebsite/HelloWorld.cshtml` или любая страница. Дополнительные сведения о публикации сайта см. ## <a name="this-is-what-aspnet-web-pages-are-about"></a>Именно это веб-страницы ASP.NET

Когда вы прочитаете эту веб-страницы ASP.NET создаете динамическое веб-содержимое, то, что вы видели здесь, это идея. Страница, которую вы только что создали, содержит ту же разметку HTML, которую вы видели ранее. Он также может содержать код, который может выполнять все виды задач. В этом примере она выполнила тривиальные задачи по получению текущей даты и времени. Как вы видели, вы можете разкодировать код с помощью HTML, чтобы получить выходные данные на странице. Когда кто-то запрашивает в браузере страницу *. cshtml* , ASP.NET обрабатывает страницу, пока она все еще находится в руки веб – сервера. ASP.NET вставляет выходные данные кода (если таковые имеются) на страницу в виде HTML. По завершении обработки кода ASP.NET отправляет полученную страницу в браузер. Весь браузер всегда получает HTML. Вот схема:

![Концептуальный поток, как ASP.NET динамически создает HTML](getting-started/_static/image21.png)

Идея проста, но существует множество интересных задач, которые может выполнять код, и существует множество интересных способов динамического добавления HTML-содержимого на страницу. И страницы ASP.NET *. cshtml* , как и любая HTML-страница, могут также включать код, выполняемый в самом браузере (код JavaScript и JQuery). Все эти действия будут рассмотрены в этом руководстве и последующих.
